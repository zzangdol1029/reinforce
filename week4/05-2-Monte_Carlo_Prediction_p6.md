## 1. 핵심 개념: State-Value Function (상태 가치 함수)

### 정의

$$v_\pi(s) = \mathbb{E}[G \mid s]$$

| 기호 | 의미 |
|------|------|
| $\pi$ | 에이전트가 따르는 **정책** |
| $s$ | **시작 상태** |
| $G$ | 그 상태에서 출발해 정책 $\pi$로 행동했을 때 얻는 **수익(return)** |
| $v_\pi(s)$ | 위 조건에서 $G$의 **기댓값** |

**직관:** “지금 이 상태 $s$에 있으면, 앞으로 **얼마나 보상을 받을 수 있을까?**”에 대한 기댓값입니다.

---

## 2. Monte Carlo (MC) 방법이란?

**핵심 아이디어:** 환경 모델을 미리 알 필요 없이, **실제로 여러 번 에피소드를 경험**하고, 그때 관측한 수익들의 **평균**으로 가치 함수를 추정합니다.

$$V_\pi(s) \approx \frac{G^{(1)} + G^{(2)} + \cdots + G^{(n)}}{n}$$

| 기호 | 의미 |
|------|------|
| $G^{(n)}$ | **$n$번째 에피소드**에서, 상태 $s$에서 출발해 실제로 얻은 **return**(누적 보상; 슬라이드 예시에서는 할인 없이 단순 합으로 설명) |
| $n$ | 에피소드 개수 |

에피소드를 **많이** 경험할수록, 위 평균은 참값 $v_\pi(s)$에 **수렴**합니다 (대수의 법칙).

---

## 3. 슬라이드 예시: 같은 시작 상태 $s$에서 두 번의 에피소드

### 에피소드 구조 (트리 다이어그램 기호)

| 기호 | 의미 |
|------|------|
| ● (검은 점) | **행동 선택** 노드 (에이전트가 $\pi$에 따라 행동 선택) |
| ○ (원) | **환경의 확률적 전이** 노드 (다음 상태·보상이 확률적으로 결정) |
| □ (사각형) | **종료 상태** (terminal state) |
| 숫자 | 각 단계에서 받은 **즉시 보상** $r$ |

(슬라이드의 그림은 위 기호로 그려진 **에피소드 트리**입니다.)

### 첫 번째 에피소드 (첫 번째 시도)

받은 보상 순서: **1 → 0 → 2**

$$G^{(1)} = 1 + 0 + 2 = \mathbf{3}$$

### 두 번째 에피소드 (두 번째 시도)

받은 보상 순서: **0 → 0 → 1 → 1**

$$G^{(2)} = 0 + 0 + 1 + 1 = \mathbf{2}$$

### 두 에피소드의 평균

$$\frac{G^{(1)} + G^{(2)}}{2} = \frac{3 + 2}{2} = \mathbf{2.5}$$

➡️ 따라서 상태 $s$의 가치를 **2.5**로 추정합니다. 이것이 **학습(learning)**입니다.

> **참고:** 일반적으로 return은 $G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots$ 처럼 **할인**이 들어갑니다. 슬라이드 6페이지의 숫자 예시는 설명을 위해 **$\gamma = 1$인 단순 합**으로 적힌 경우가 많습니다.

## 4. Dynamic Programming (DP)과의 비교

| 구분 | Monte Carlo (MC) | Dynamic Programming (DP) |
|------|------------------|---------------------------|
| **방법** | 실제 **경험(에피소드)**의 평균 | **수식**으로 직접 계산 (벨만 기대 방정식 등) |
| **핵심** | **Learning** (학습) | **Computing** (계산) |
| **필요한 것** | 에피소드 경험만 있으면 됨 | **환경 모델** $p(s' \mid s, a)$ (전이 확률) 필요 |

### MC 쪽 수식 (개념)

$$V_\pi(s) \approx \frac{\sum_n G^{(n)}}{n}$$

### DP 쪽 수식 (정책 평가, 벨만 기대 방정식)

$$V_{k+1}(s) = \sum_{a,s'} \pi(a \mid s)\, p(s' \mid s, a)\,\bigl\{ r(s,a,s') + \gamma V_k(s') \bigr\}$$

#### DP 수식 항목 해석

| 항 | 의미 |
|----|------|
| $\pi(a \mid s)$ | 정책에 따라 상태 $s$에서 행동 $a$를 선택할 **확률** |
| $p(s' \mid s, a)$ | 행동 $a$ 후 다음 상태가 $s'$가 될 **전이 확률** (**환경 모델**) |
| $r(s,a,s')$ | 그 전이에서의 **즉시 보상** |
| $\gamma V_k(s')$ | 다음 상태 $s'$의 가치를 **할인**한 항 |

---

## 5. 핵심 요약

1. **Monte Carlo**는 **환경 모델 없이도** 실제 경험만으로 가치 함수를 **학습**할 수 있다는 점이 가장 큰 장점입니다.

2. **DP**는 전이 확률 $p(s' \mid s, a)$ 등을 **알아야** 하지만, **MC**는 몰라도 됩니다.

3. **한계:** MC는 **에피소드가 끝난 뒤**에야 return $G$를 확정할 수 있으므로, **에피소드가 반드시 끝나는 문제**(episodic)에 적합합니다. (진행 중인 연속 과제에는 그대로 쓰기 어렵습니다.)
