## 전체 목표

**고정된 정책 π**를 따랐을 때, 각 상태 $s$에 대한 **상태 가치** $V_\pi(s)$를 **에피소드 경험만으로** 추정한다.

$$\text{입력: 정책 } \pi \quad \Rightarrow \quad \text{출력: } V \approx V_\pi$$

---

## 초기화 (Initialize)

| 기호 | 의미 | 비유 |
|------|------|------|
| **π** | 평가할 **정책** (어떤 행동을 할지 정해 둔 규칙) | 게임 **공략집** |
| **V(s)** | 각 상태의 가치 **추정값** (초기값은 임의, 보통 0으로 둠) | **점수판** (처음엔 다 0) |
| **Returns(s)** | 상태 $s$에서 관측한 return $G$들을 모아 두는 **리스트** | 상태별 **성적 기록부** |

```
π ← 평가할 정책
V ← 임의의 상태 가치 함수 (아무 값이나 초기화)
Returns(s) ← 빈 리스트  (모든 상태 s에 대해)
```

---

## 반복 (Repeat forever)

### Step 1 — 에피소드 생성

정책 **π**에 따라 **에피소드 하나**를 **끝까지** 진행한다.

```
예시:  A → B → C → 종료
       R₀   R₁   R₂   (각 전이에서 받은 보상)
```

### Step 2 — 각 상태마다 G 계산 후 기록

에피소드에 **등장한 각 상태** $s$에 대해:

1. **G** ← 그 상태에서 **첫 등장 이후** 에피소드 끝까지의 총 return (First-Visit; 아래 참고)
2. **Returns(s)**에 G를 **추가**
3. **V(s)** ← **Returns(s)의 평균**

#### 숫자 예시 (상태 A만 추적)

| 에피소드 | 관측한 $G_A$ | Returns(A) | V(A) = 평균 |
|----------|--------------|------------|-------------|
| 1번째 | 3 | [3] | **3.0** |
| 2번째 | 2 | [3, 2] | **(3+2)/2 = 2.5** |
| 3번째 | 4 | [3, 2, 4] | **(3+2+4)/3 = 3.0** |

에피소드가 쌓일수록 $V(s)$는 참값 $v_\pi(s)$에 **수렴**한다 (대수의 법칙).

---

## 최종 수식과의 연결

알고리즘이 하는 일을 한 줄로 쓰면:

$$V_\pi(s) \approx \frac{G^{(1)} + G^{(2)} + \cdots + G^{(n)}}{n}$$

- $G^{(k)}$: $k$번째 에피소드에서 상태 $s$에 대해 기록한 return  
- $n$: 그 상태에 대해 쌓인 샘플 개수  

즉 **Returns(s)의 원소들의 평균**이 $V(s)$이다.

---

## 전체 흐름 요약

```
에피소드 실행 (π로 끝까지)
        ↓
각 상태의 G 계산 (역방향 누적 등으로 효율적으로)
        ↓
Returns(s)에 G 추가
        ↓
V(s) = Returns(s)의 평균 으로 갱신
        ↓
다시 에피소드 실행 … (무한 반복)
        ↓
에피소드가 많아질수록 V(s) → 진짜 v_π(s)에 수렴 ✅
```

---

## 핵심 포인트: First-Visit Monte Carlo

> **"first occurrence of s"** — 한 에피소드 안에서 **같은 상태 s가 여러 번** 나올 수 있다.

이 알고리즘(슬라이드·교재에서 흔히 소개하는 형태)은 **그 에피소드에서 s가 처음 등장한 시점**부터 끝까지의 return **하나만** 사용한다.

- 이렇게 쓰는 방법을 **First-Visit MC**라고 부른다.
- 반대로 같은 에피소드에서 s가 나올 때마다 return을 모두 쓰면 **Every-Visit MC** 등 다른 변형이 된다.

`mc_eval.py`의 `RandomAgent`는 에피소드당 한 번씩만 각 (state, …) 구간을 역순으로 돌며 업데이트하는 구조라, **한 경로에서 상태가 한 번씩만 등장**하는 단순한 격자 예시와 맞물려 First-Visit 아이디어와 잘 연결된다.
