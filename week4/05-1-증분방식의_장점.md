# 증분 방식(Incremental Update)의 장점

> 이 문서는 [`05-Monte_Carlo_Method.md`](./05-Monte_Carlo_Method.md) **1.5절 몬테카를로 업데이트 공식**의 심화 설명입니다.

---

## 핵심 질문

> **왜 모든 결과를 다 모아서 나누는 방식 대신, 굳이 증분 방식을 쓰는가?**

주사위를 정해진 횟수(예: 1,000번)만큼 반복하는 것은 **두 방식 모두 동일**합니다.  
결정적인 차이는 **"계산 횟수"가 아니라** 다음 두 가지입니다.

1. 컴퓨터의 **메모리(기억 공간)를 어떻게 쓰느냐**
2. **언제 계산하느냐** (마지막에 한꺼번에 vs 매 순간 실시간으로)

---

## 두 방식 비교

### 방식 1 — 일반 평균 (모아서 나누기)

```python
samples = []                      # 결과를 담을 리스트 (크기 무제한으로 계속 커짐)
for i in range(n):
    s = throw_dice()
    samples.append(s)             # 던질 때마다 결과를 리스트에 누적

V = sum(samples) / len(samples)   # 마지막에 한꺼번에 계산
```

$$V_n = \frac{s_1 + s_2 + \cdots + s_n}{n}$$

**문제점:**

| 횟수 | 메모리에 보관해야 할 숫자 개수 |
|------|-------------------------------|
| 1,000번 | 1,000개 |
| 100만 번 | 100만 개 |
| 1억 번 | **1억 개** ← 컴퓨터가 무거워짐 |

- 1,000번을 다 던지기 전까지는 **"현재 평균이 얼마인지" 알 수 없음**
- 모든 데이터가 다 모일 때까지 **기다려야 함**

---

### 방식 2 — 증분 방식 (그때그때 업데이트)

```python
V = 0.0    # 현재까지의 평균값  ← 기억할 숫자 1
n = 0      # 현재까지 던진 횟수 ← 기억할 숫자 2

for i in range(1000):
    s = throw_dice()           # 새 주사위 결과
    n += 1
    V = V + (s - V) / n        # 평균만 살짝 수정하고, s는 바로 잊어버림
```

$$\boxed{V_n = V_{n-1} + \frac{1}{n}(s_n - V_{n-1})}$$

**기억해야 할 숫자는 단 2개:**

| 변수 | 의미 |
|------|------|
| `V` | 지금까지의 **평균값** (현재 추정치) |
| `n` | 지금까지 **던진 횟수** |

- 새 결과 `s`가 오면 `V`를 살짝 업데이트하고, **`s`는 즉시 버림**
- 주사위를 **1억 번** 던져도 기억해야 할 숫자는 **항상 2개뿐**

---

## 장점 1 — 압도적인 메모리 절약 (Memory Efficiency)

```
방식 1:  ■■■■■■■■■■■■■■■■■■■■  ... (n개 저장)
방식 2:  ■■                         (항상 2개만 유지)
```

- 과거에 던졌던 모든 결과를 **하나도 저장하지 않음**
- 데이터가 아무리 많아져도 메모리 사용량은 **상수(constant)** 로 고정
- 스트리밍 데이터, 실시간 센서 값처럼 **끝이 없는 데이터**에도 적용 가능

---

## 장점 2 — 실시간 업데이트 (Online Learning)

| | 방식 1 (일반 평균) | 방식 2 (증분) |
|-|-------------------|--------------|
| 현재 추정값 확인 | ❌ 마지막에만 가능 | ✅ **매 순간 확인 가능** |
| 새 데이터 반영 | ❌ 모두 모인 후 | ✅ **즉시 반영** |
| 중간에 멈춰도 | ❌ 결과 없음 | ✅ **그 시점까지의 추정값 존재** |

주사위를 1번 던질 때마다 **현재까지의 최선의 추정값**이 항상 갱신되어 있습니다.

## 강화학습에서의 의미

에이전트(로봇, AI)는 수없이 많은 **에피소드(경험)**를 쌓으며 학습합니다.

### 방식 1을 강화학습에 쓰면?

```
로봇이 태어나서 → 죽을 때까지의 모든 (행동, 보상) 쌍을 전부 기억 →
그래야만 가치 평가 가능
```

→ 에피소드가 쌓일수록 메모리가 폭발적으로 증가, 현실적으로 불가능

### 방식 2 (증분)를 강화학습에 쓰면?

```
에피소드 1 → V 업데이트 → 에피소드 1 내용 잊어버림
에피소드 2 → V 업데이트 → 에피소드 2 내용 잊어버림
에피소드 3 → V 업데이트 → 에피소드 3 내용 잊어버림
...
```

→ 로봇은 **"지금까지의 평균적인 가치(V)"** 만 머릿속에 가볍게 유지하면서,  
새 경험이 생길 때마다 **즉시 자신의 지식을 업데이트(학습)**

이것이 바로 `mc_eval.py`의 `eval()` 메서드가 하는 일입니다:

```python
def eval(self):
    G = 0.0
    for (state, action, reward) in reversed(self.memory):
        G = self.gamma * G + reward          # 이번 에피소드의 수익 G 계산
        self.cnts[state] += 1
        self.V[state] += (G - self.V[state]) / self.cnts[state]
        #   ↑ V와 cnts 두 값만 업데이트. 이 에피소드의 세부 기록은 eval() 이후 버려짐
```

## 한 눈에 보는 비유

| | 방식 1 | 방식 2 (증분) |
|-|--------|--------------|
| **비유** | 일기장에 모든 하루를 다 적어두고, 마지막 날 전부 읽어서 평균을 냄 | 매일 밤 "지금까지의 나의 평균 하루 점수"만 수첩에 적어두고 어제 일기는 버림 |
| **장점** | 과거 데이터를 언제든 다시 볼 수 있음 | 공간 절약, 실시간 업데이트 |
| **강화학습 적합성** | ❌ 비현실적 | ✅ 핵심 원리로 사용됨 |

## 정리

> **증분 방식의 핵심:** "과거의 구체적인 기억은 잊어버려도, '현재까지의 짬바(평균적인 가치)'만 유지한 채 새 경험이 올 때마다 즉시 업데이트한다."

$$V_n = V_{n-1} + \frac{1}{n}(s_n - V_{n-1})$$

- `V_{n-1}`: 지금까지의 평균 (과거 경험 전체가 이 숫자 한 개에 압축)
- `s_n - V_{n-1}`: 새 결과와 기존 평균의 **오차(error)**
- `1/n`: 오차를 **얼마나 반영할지**의 비율 (나중에 이것이 고정값 α로 대체됨 → 지수 이동 평균)
