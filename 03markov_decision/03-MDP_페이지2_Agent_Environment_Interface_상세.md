# 03-Markov Decision Process.pdf — **2페이지** 상세 설명  
## Agent–Environment Interface (에이전트–환경 인터페이스)

> 강의 슬라이드 2페이지: *The agent-environment interaction in RL*  
> 본 문서는 2페이지에 나오는 용어·기호·개념을 **최대한 풀어서** 정리합니다.

---

## 1. 2페이지가 말하는 한 줄 요약

**강화학습(RL)**에서는 **에이전트(Agent)**와 **환경(Environment)**이 시간에 따라 반복적으로 상호작용합니다.  
환경은 **상태**와 **보상**을 주고, 에이전트는 **정책**에 따라 **행동**을 선택합니다.  
RL의 목표는 **누적 보상(또는 할인된 보상의 합)**이 최대가 되도록 **정책을 정하는 것**입니다.

---

## 2. 왜 “인터페이스(Interface)”라고 부를까?

**인터페이스**는 두 시스템이 **서로 주고받을 수 있는 정보의 규약**을 뜻합니다.

| 방향 | 무엇이 오가는가 |
|------|-----------------|
| **환경 → 에이전트** | 다음 **상태** \(S_{t+1}\), **보상** \(R_{t+1}\) (슬라이드 표기에 따라 인덱스는 \(t\) 또는 \(t+1\)로 다를 수 있음) |
| **에이전트 → 환경** | **행동** \(A_t\) |

즉, RL에서 “무엇을 관측하고, 무엇을 할 수 있으며, 무엇을 피드백으로 받는가”가 **문제 정의의 핵심**입니다.  
이 규약이 정해져야 MDP, 알고리즘(DQN, PPO 등)을 적용할 수 있습니다.

---

## 3. Agent (에이전트)

### 3.1 슬라이드 정의

- **Agent**: *learner, decision-maker (controller)*  
  - **학습자(learner)**: 경험을 통해 정책을 바꿀 수 있음  
  - **의사결정자(decision-maker)**: 매 시점 어떤 행동을 할지 고름  
  - **제어기(controller)**: 공학적으로는 플랜트(환경)에 **제어 입력(행동)**을 넣는 주체

### 3.2 직관적 예시

| 문제 | 에이전트 |
|------|----------|
| 게임 | 플레이어 또는 AI 플레이어 |
| 로봇 팔 | 제어 소프트웨어 |
| 추천 시스템 | 추천 알고리즘 |
| 자율주행 | 주행 정책 모듈 |

에이전트는 **환경의 내부 구조를 모를 수도**(model-free), **모델을 가질 수도**(model-based) 있지만,  
2페이지 관점에서는 “**상태를 보고 행동을보내는 블록**”으로 보면 됩니다.

---

## 4. Environment (환경)

### 3.1 슬라이드 정의

- **Environment**: *The thing which the agent interact with (plant or system)*  
  - 에이전트와 **상호작용하는 대상 전체**  
  - 공학에서는 **plant**(제어 대상), 일반적으로는 **system**(시스템)

### 4.2 환경이 하는 일 (역할)

1. 에이전트의 행동 \(A_t\)를 받음  
2. (내부 규칙에 따라) 다음 상태 \(S_{t+1}\)와 보상 \(R_{t+1}\)을 생성  
3. 에이전트에게 그 정보를 넘겨줌  

**중요**: 환경은 “세상 전부”가 아니라 **모델링 경계**입니다.  
예: 로봇의 경우, **로봇 본체의 동역학 + 외부 장애물**까지 환경에 넣을지, **센서만** 상태로 둘지는 **설계 선택**입니다.

---

## 5. State (상태) — \(S_t \in \mathcal{S}\)

### 5.1 기호

- \(S_t\): 시각 \(t\)에서의 **상태** (random variable로 취급)  
- \(\mathcal{S}\): 가능한 모든 상태의 집합 (**state space**)

### 5.2 의미

상태는 **의사결정에 필요한 정보를 담은 요약**입니다.

- 이상적으로는 **Markov 성질**을 만족: “현재 상태만 알면 미래가 과거와 조건부 독립” (이후 슬라이드에서 정식화)  
- 실무에서는 완벽한 Markov가 아니면 **근사**하거나 **히스토리를 상태에 포함**(RNN, frame stack 등)

### 5.3 예시

| 문제 | 상태에 들어갈 수 있는 것 |
|------|---------------------------|
| 격자 세계 | 현재 칸 좌표 |
| 카트폴 | 카트 위치, 속도, 막대 각도, 각속도 |
| 보드게임 | 말 배치, 턴, 규칙에 따른 추가 정보 |

2페이지에서는 “상태가 있다”는 점과 **집합 \(\mathcal{S}\)** 까지만 강조합니다.

---

## 6. Action (행동) — \(A_t \in \mathcal{A}(S_t)\)

### 6.1 기호

- \(A_t\): 시각 \(t\)에서 선택한 **행동**  
- \(\mathcal{A}(S_t)\): **현재 상태 \(S_t\)에서 선택 가능한 행동 집합**

### 6.2 왜 \(\mathcal{A}(S_t)\) 인가?

행동 가능 집합이 **상태에 따라 달라질 수 있기 때문**입니다.

- 특정 상태에서는 **일부 행동이 불가** (예: 벽 앞에서 “앞으로” 불가)  
- 게임에서 **스킬 쿨타임** 등으로 행동 집합이 변함  

따라서 일반적으로 \(\mathcal{A}\) 하나로 고정하기보다 **\(\mathcal{A}(s)\)** 로 쓰는 것이 정확합니다.

### 6.3 이산 vs 연속

- **이산(discrete)**: 행동 개수가 유한 — DQN, Q-learning에 흔함  
- **연속(continuous)**: 토크, 가속도 등 실수 벡터 — DDPG, SAC 등

2페이지는 이 구분을 명시하진 않지만, **인터페이스** 관점에서 “행동 공간의 형태”는 문제마다 다릅니다.

---

## 7. Reward (보상) — \(R_{t+1} \in \mathcal{R} \subset \mathbb{R}\)

### 7.1 기호

- 보상은 **실수** 스칼라: \(\mathcal{R} \subset \mathbb{R}\)  
- 시점 표기는 교재마다 \(R_t\) 또는 \(R_{t+1}\) 로 다름 — **같은 내용을 다른 인덱스로 쓴 것**인 경우가 많음

### 7.2 역할

- 에이전트에게 “**원하는 행동**”을 **숫자**로 알려주는 신호  
- RL 알고리즘은 이 신호를 모아 **장기 누적**이 커지도록 학습

### 7.3 설계 시 주의 (2페이지 이후와 연결)

- **희소(sparse) 보상**: 자주 0, 가끔만 큰 보상 → 탐색이 어려움  
- **지연(delayed) 보상**: 나중에야 좋은지 알게 됨 → 신용 할당(credit assignment) 문제  

2페이지는 “보상이 **수치**로 온다”는 **인터페이스**만 제시합니다.

---

## 8. Policy (정책) — \(\pi_t(a|s)\)

### 8.1 슬라이드 정의

- \(\pi_t = \pi_t(a|s)\): **\(S_t = s\) 일 때 \(A_t = a\) 일 확률**  
- *mapping from states to probabilities of each possible action*  
- **deterministic / stochastic** 모두 가능

### 8.2 확률 정책 \(\pi(a|s)\)

\[
\pi(a|s) = P(A_t = a \mid S_t = s)
\]

조건: 각 상태 \(s\)에서 \(\sum_{a \in \mathcal{A}(s)} \pi(a|s) = 1\), \(\pi(a|s) \ge 0\).

**stochastic 정책**의 이유 (개념적으로):

- **탐색(exploration)**: 모든 행동에 작은 확률을 줌  
- **환경이 확률적**일 때 혼합이 최적인 경우  
- **여러 최적 행동**이 있을 때 무작위화

### 8.3 Deterministic 정책

행동 하나에 확률 1:

\[
\pi(a|s) = \begin{cases} 1 & a = \mu(s) \\ 0 & \text{otherwise} \end{cases}
\]

강의 후반(1×2 Grid 등)에서 **\(\mu(s)\)** 로 deterministic 정책을 표로 적는 것과 동일한 맥락입니다.

### 8.4 시간 의존 \(\pi_t\)

첨자 \(t\)는 **정책이 시간에 따라 바뀔 수 있음**을 허용합니다.

- 학습 초기: \(\pi_0\) (무작위에 가깝게)  
- 학습 후: \(\pi_T\) (거의 greedy)  
- 정책이 고정이면 \(\pi_t = \pi\) 로 씁니다.

---

## 9. 한 스텝 상호작용의 시간 순서 (이해용)

많은 교재에서 한 스텝을 다음처럼 둡니다.

1. 시각 \(t\): 에이전트가 상태 \(S_t\)를 관측  
2. 정책에 따라 \(A_t \sim \pi(\cdot \mid S_t)\) 선택  
3. 환경이 \(A_t\)를 받아 \(S_{t+1}, R_{t+1}\) 생성  
4. \(t \leftarrow t+1\) 반복  

**핵심**: 에이전트는 **과거 전체가 아니라 (보통) 현재 상태(또는 관측)**를 입력으로 행동을 고릅니다.  
(부분 관측이면 \(S_t\) 대신 관측 \(O_t\) — 이는 POMDP로 확장.)

---

## 10. 2페이지의 결론 문장 해석

> **RL determines agent’s policy to maximize the total amount of rewards**

- **무엇을 학습/결정하는가?** → 에이전트의 **정책 \(\pi\)**  
- **무엇을 최대화하는가?** → **보상의 총합**(또는 할인합; 다음 슬라이드에서 Return·\(\gamma\)로 정식화)

즉 2페이지는 **목적 함수의 방향**을 한 문장으로 고정합니다.  
“상태 가치가 크다”는 말도 결국 **그 상태에서 출발해 얻는 기댓값 보상의 합**을 말하는 것입니다.

---

## 11. 용어·기호 빠른 참조표 (2페이지 전용)

| 기호/용어 | 의미 |
|-----------|------|
| Agent | 학습·의사결정·제어 주체 |
| Environment | 에이전트와 맞닿은 외부 세계(모델링 범위) |
| \(S_t \in \mathcal{S}\) | 시각 \(t\)의 상태, 상태 공간 \(\mathcal{S}\) |
| \(A_t \in \mathcal{A}(S_t)\) | 시각 \(t\)의 행동, 상태 의존 행동 집합 |
| \(R_{t+1}\) | (스텝 후) 보상, 실수 |
| \(\pi_t(a|s)\) | 상태 \(s\)에서 행동 \(a\)를 택할 확률 |
| deterministic \(\pi\) | 한 상태에 대해 행동 하나만 확률 1 |
| stochastic \(\pi\) | 한 상태에서 여러 행동에 확률 분배 |

## 12. 2페이지와 이후 슬라이드의 연결

| 이후 주제 | 2페이지와의 관계 |
|-----------|-------------------|
| Return, \(\gamma\) | “total amount of rewards”를 **무한 합에서도 잘 정의**하기 위함 |
| Markov property | \(S_t\)만으로 다음이 결정된다는 **가정** |
| MDP | \(\mathcal{S}, \mathcal{A}, p(s',r\mid s,a), \gamma\) 로 환경 역학을 정식화 |
| Value function | 정책 \(\pi\) 아래 **기댓값 보상의 합**을 상태/행동별로 측정 |
| Bellman equation | 가치를 **재귀적으로** 연결 — 여전히 같은 인터페이스 위에서 성립 |

---

## 13. 스스로 확인하면 좋은 질문 (자기점검)

1. 내가 풀려는 문제에서 **상태**는 무엇으로 둘 것인가? \(\mathcal{S}\)는 유한한가?  
2. **행동 집합**은 모든 상태에서 동일한가? 아니면 \(\mathcal{A}(s)\) 인가?  
3. **보상**은 언제, 무엇에 대해 주는가? (희소/지연은?)  
4. 정책은 **확률적**으로 둘 필요가 있는가, **결정적**으로 충분한가?  
5. “환경”의 경계를 어디까지 둘 것인가? (센서만 vs 물리 전체)

---

## 14. 한 문단으로 압축 (암기용)

에이전트는 매 단계 **상태**를 보고 **정책**에 따라 **행동**을 고르고, 환경은 **보상**과 다음 **상태**를 돌려준다. 강화학습은 이 상호작용을 통해 **누적 보상이 최대**가 되도록 **정책**을 고르거나 학습하는 것이다.
