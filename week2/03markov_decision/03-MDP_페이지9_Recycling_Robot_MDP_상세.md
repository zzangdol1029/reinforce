## Recycling Robot MDP — 상태·행동·전이 표

> 강의 슬라이드 9페이지: Recycling Robot의 **상태 공간**, **행동 공간**, **전이 확률·기대 보상** 표  
> 본 문서는 **8페이지**에서 배운 \(p(s'|s,a)\), \(r(s,a,s')\) 등을 **이 예제에 어떻게 대응**하는지 풀어 씁니다.  
> (슬라이드의 **정확한 수치·기호**는 본인 PDF가 정본입니다. 아래 \(\alpha,\beta,r_{\text{search}},r_{\text{wait}},-3\) 등은 전형적인 교재 설정과 사용자 제공 설명을 반영했습니다.)

---

## Recycling Robot, 쉬운 말로 (최상단)

**재활용 로봇**은 **배터리**(high / low)에 따라 **할 수 있는 일**이 달라지고, **탐색·대기·충전**을 하면 **다음 배터리 상태**와 **캔 수집 보상**이 **확률적으로** 정해집니다.

**9페이지**의 **복잡해 보이는 표**는, **8페이지**의

- **전이 확률** \(p(s' \mid s, a)\)  
- **(상태, 행동, 다음상태)에 대한 기대 보상** \(r(s, a, s')\)  

을 **모든 \((s,a,s')\) 경우**에 대해 **한눈에** 모아 둔 것입니다.

**한 줄**: \(\mathcal{S}=\{\text{high},\text{low}\}\), 배터리에 따라 \(\mathcal{A}(s)\) 가 다르고, **wait / recharge / search** 마다 **다음 상태 확률**과 **보상**이 표에 정리되어 있다.

---

## 기호 빠른 정의 (9페이지용)

| 기호 | 의미 (이 슬라이드 맥락) |
|------|-------------------------|
| **high / low** | 배터리 **충분** / **부족** |
| **search** | 빈 캔을 **적극 탐색** (에너지 소모, 확률적 전이) |
| **wait** | **대기** (배터리 거의 유지, 보상 \(r_{\text{wait}}\) ) |
| **recharge** | **충전** (low에서만, high로 복귀) |
| **\(\alpha\)** | **high**에서 **search** 시, 다음에도 **high** 유지될 확률 (\(1-\alpha\) 로 **low** 전이) |
| **\(\beta\)** | **low**에서 **search** 시, 다음에도 **low** 유지될 확률 (\(1-\beta\) 로 **방전** 등 특수 전이) |
| **\(r_{\text{search}}\)** | 탐색 중 **평균적으로** 얻는 캔 관련 보상(교재에서 하나의 기댓값으로 둠) |
| **\(r_{\text{wait}}\)** | 대기 중 **평균적으로** 얻는 캔 관련 보상 |
| **\(-3\)** (예시) | low에서 search 중 **심한 방전** 등으로 인한 **큰 페널티** (슬라이드 수치 따름) |

## 1. 9페이지가 하는 일 (강의 흐름)

- **8페이지**: MDP 튜플, \(p(s',r|s,a)\), \(p(s'|s,a)\), 기댓값 보상의 **일반 공식**  
- **9페이지**: **Recycling Robot** 하나를 골라 \(\mathcal{S},\mathcal{A}\), **표**로 **완전한 한 스텝 역학**을 채움  
- 이후: 전이 **그래프** 그리기, Bellman 식 대입 등으로 연결  

즉 **“공식을 숫자·확률로 채운 예”** 페이지입니다.

---

## 2. 상태 공간 \(\mathcal{S}\)

로봇이 인식하는 **현재 상태**를 **배터리 잔량의 이산 요약**으로 둡니다.

\[
\mathcal{S} = \{\text{high},\ \text{low}\}
\]

- **high**: 배터리가 **충분**해서 적극 활동 가능한 구간  
- **low**: 배터리가 **부족**해서 위험하거나 충전이 필요한 구간  

(실제 전압을 연속값으로 두지 않고 **두 구간만** 쓰는 **단순화**입니다.)

---

## 3. 행동 공간 \(\mathcal{A}(s)\) — 상태마다 다름

로봇이 **취할 수 있는 행동**은 **배터리 상태에 따라 달라집니다**. (8페이지의 \(\mathcal{A}(s)\) 개념 그대로입니다.)

### 3.1 배터리가 high일 때

\[
\mathcal{A}(\text{high}) = \{\text{search},\ \text{wait}\}
\]

- **search**: 능동적으로 빈 캔을 **찾아다님** (탐색)  
- **wait**: 누군가 캔을 가져다주기를 **기다림** (대기)  

**high에서는 recharge가 없음** — 이미 배터리가 충분하므로 **충전 행동을 모델에 넣지 않는** 전형적 설정입니다.

### 3.2 배터리가 low일 때

\[
\mathcal{A}(\text{low}) = \{\text{search},\ \text{wait},\ \text{recharge}\}
\]

- **search**, **wait** 는 high와 동일한 **이름**이지만, **전이·보상 표**가 다름 (위험도가 다름)  
- **recharge**: **홈 베이스** 등으로 돌아가 **배터리 충전** — **low에서만** 선택 가능  

---

## 4. 9페이지 표가 의미하는 것 — 8페이지 공식과 연결

슬라이드 표의 각 행은 보통 다음 정보를 담습니다.

- **현재 상태 \(s\)**, **행동 \(a\)**, **다음 상태 \(s'\)**  
- **그 전이가 일어날 확률** → \(p(s' \mid s, a)\) 의 **한 조각** (같은 \((s,a)\)에 대해 \(s'\)별 확률의 **합이 1**)  
- **그 \((s,a,s')\) 에서의 기대 보상** → \(r(s, a, s')\) (또는 결합 \(p(s',r|s,a)\) 를 압축해 둔 값)

**정리**: 표는 \(p(s', r \mid s, a)\) 를 풀어 쓴 것이거나, **\(p(s'|s,a)\)** 와 **\(r(s,a,s')\)** 를 **나란히** 적어 둔 형태입니다.

---

## 5. 행동별 분석

### A. 대기 (Wait)

**특징**: **가장 단순·안전**한 행동으로 많이 모델링됩니다.

- **high** 또는 **low** **어느 상태에서든** 선택 가능  
- 배터리를 **거의 소모하지 않는다**고 가정하면, **다음 상태**는 **현재와 동일**할 **확률 1**:

\[
p(\text{high} \mid \text{high}, \text{wait}) = 1,\qquad
p(\text{low} \mid \text{low}, \text{wait}) = 1
\]

- **보상**: 대기하는 동안 **평균적으로** 얻는 캔의 양(또는 그에 해당하는 점수)을 **\(r_{\text{wait}}\)** 하나로 둡니다.  
  즉 \(r(\text{high},\text{wait},\text{high}) = r_{\text{wait}}\), \(r(\text{low},\text{wait},\text{low}) = r_{\text{wait}}\) 처럼 표에 적힙니다.

---

### B. 충전 (Recharge)

**선택 가능 조건**: **low** 에서만 (\(\mathcal{A}(\text{low})\) 에만 포함).

- 충전을 선택하면 **다음 상태는 high** 로 **복구**된다고 두는 경우가 많고, **확률 1**:

\[
p(\text{high} \mid \text{low}, \text{recharge}) = 1
\]

- 충전하러 가는 동안에는 **캔 수집을 하지 못한다**고 두면 **보상 0**:

\[
r(\text{low},\text{recharge},\text{high}) = 0
\]

---

### C. 탐색 (Search) — **확률적** 전이

**특징**: **적극적**이지만 **배터리 소모**로 **상태가 바뀔 확률**이 들어갑니다.

공통적으로, 탐색 중 **평균적으로** 주운 캔에 해당하는 보상을 **\(r_{\text{search}}\)** 로 둡니다 (세부 \(s'\) 마다 슬라이드가 같거나 다르게 둘 수 있음).

#### C.1 배터리가 **high** 일 때 search

- 에너지를 써도 **여전히 high** 일 확률: **\(\alpha\)**  
  \[
  p(\text{high} \mid \text{high}, \text{search}) = \alpha
  \]
- **많이 써서 low** 로 떨어질 확률: **\(1-\alpha\)**  
  \[
  p(\text{low} \mid \text{high}, \text{search}) = 1-\alpha
  \]

두 확률의 합은 **1**입니다. 각 가지에 대한 **\(r_{\text{search}}\)** 는 교재·슬라이드가 동일하게 두는지 확인하면 됩니다.

#### C.2 배터리가 **low** 일 때 search (위험)

**아슬아슬한** 구간으로 모델링됩니다.

- 탐색을 **무사히** 마치고 **low 유지**될 확률: **\(\beta\)**  
  \[
  p(\text{low} \mid \text{low}, \text{search}) = \beta
  \]
- 탐색 중 **심하게 방전** 등으로 **특수한 결과**가 나올 확률: **\(1-\beta\)**  
  - 슬라이드·교재에 따라 “구출되어 **high**로 돌아간다” 등 **다음 상태 \(s'\)** 를 명시합니다.  
  - 이때 **큰 페널티** 예: **\(-3\)** (사용자 제공 설명; **PDF 숫자가 정본**)

즉 **low + search** 에서는

- 한 가지 \(s'\) 로는 **\(r_{\text{search}}\)** (평균 수집)  
- 다른 \(s'\) 로는 **\(-3\)** 같은 **벌점**  

처럼 **\(r(s,a,s')\) 가 \(s'\)에 따라 다르게** 표에 나오는 것이 자연스럽습니다.

## 6. 표를 스스로 검산할 때 체크리스트

각 **\((s, a)\)** 에 대해:

1. 나열된 모든 **\(s'\)** 에 대한 **\(p(s'|s,a)\)** 의 **합이 1**인가?  
2. **high**에서 **recharge** 가 표에 **없어야** 정상인가? (행동 집합과 일치)  
3. **wait** 는 **자기 자신으로** 확률 1인가?  
4. **recharge** 는 **low → high**, 확률 1인가?

## 8. 요약 (암기용)

1. \(\mathcal{S} = \{\text{high}, \text{low}\}\)  
2. \(\mathcal{A}(\text{high}) = \{\text{search}, \text{wait}\}\), \(\mathcal{A}(\text{low}) = \{\text{search}, \text{wait}, \text{recharge}\}\)  
3. **wait**: 상태 유지, 확률 1, 보상 \(r_{\text{wait}}\)  
4. **recharge**: low → high, 확률 1, 보상 0 (전형적)  
5. **search**: **확률적** 전이 — high에서 \(\alpha, 1-\alpha\); low에서 \(\beta, 1-\beta\) 및 페널티 등
