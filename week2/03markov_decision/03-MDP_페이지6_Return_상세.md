# 03-Markov Decision Process.pdf — **6페이지** 상세 설명  
## Return (수익)과 할인율 \(\gamma\)

> 강의 슬라이드 6페이지: *Return* — 에이전트의 목표, \(G_t\) 정의, Episodic/Continuing 구분, discount rate \(\gamma\)  
> 본 문서는 6페이지 개념을 **최대한 풀어서** 정리합니다.

---

## 1. 6페이지가 하는 일 (강의 흐름상 위치)

- **2–4페이지**: Agent–Environment, 상태·행동·보상  
- **5페이지**: Episodic vs Continuing **과제 유형**  
- **6페이지**: 그 과제 안에서 “**얼마나 잘 했는가**”를 숫자 하나로 잴라고 할 때 쓰는 **Return \(G_t\)** 와, Continuing에서 쓰는 **할인율 \(\gamma\)**  

즉, **보상 \(R\)** 은 매 스텝마다 조금씩 오고, RL의 목표는 **그것들을 어떻게 합친 값**을 크게 하는 것인데, 그 “합친 값”의 정의가 **6페이지**입니다.

---

## 2. 에이전트의 목표 (Goal)

### 슬라이드 문장

> **Agent’s goal is to maximize the cumulative reward it receives in the long run**

**직역**: 에이전트의 목표는 **장기적으로 받는 보상을 누적한 것**을 최대화하는 것이다.

### 풀어서

- 한 번의 행동으로 끝나는 게 아니라, **시간이 흐르며** 보상이 **여러 번** 쌓입니다.  
- “잘한다”의 기준은 **지금 당장 보상만**이 아니라, **앞으로도 포함한 전체**입니다.  
- 그 “전체”를 수학적으로 쓴 것이 **Return \(G_t\)** 입니다.

**주의**: “누적”의 **정확한 식**은 과제가 **Episodic**인지 **Continuing**인지에 따라 달라집니다 (다음 절).

---

## 3. Return \(G_t\) (수익)이란?

### 3.1 직관

- 시점 \(t\)에서 **눈앞의 한 스텝**만 보는 게 아니라,  
  **\(t\) 이후에 받게 될 보상들의 합(또는 할인합)** 을 생각합니다.  
- 그 **미래 보상들을 모아 놓은 값**이 **\(t\)에서의 Return** \(G_t\) 입니다.

### 3.2 기호

- \(R_{t+1}, R_{t+2}, \ldots\): 시점 \(t\) 직후, 그다음, … 에 **받는 보상**  
  (교재마다 \(R_t\)로 쓰기도 하지만, **같은 내용을 인덱스만 다르게** 쓴 경우가 많습니다.)

---

## 4. Episodic task (일회성 과제)에서의 Return

### 4.1 슬라이드 식

\[
G_t = R_{t+1} + R_{t+2} + R_{t+3} + \cdots + R_T
\]

- **\(T\)**: 에피소드가 **끝나는** 마지막 시각(terminal에 도달하는 등)  
- **유한 합**: 항의 개수가 **유한**이므로, **발산하지 않음** (보상이 유한이라면).

### 4.2 직관

- 게임 한 판, 격자에서 목표까지 한 번 가는 것처럼 **끝이 있는** 문제  
- \(t\)에서 출발해 **남은 플레이**에서 받는 보상을 **다 더함**  
- **할인 \(\gamma\)** 를 곱하지 않는 **단순 합**으로 두는 경우가 많음 (슬라이드가 그 형태)

### 4.3 작은 숫자 예시

\(T=5\)이고, \(t=2\)에서 남은 보상이 \(R_3=1, R_4=0, R_5=-1\) 이라면:

\[
G_2 = R_3 + R_4 + R_5 = 1 + 0 + (-1) = 0
\]

\(t=4\)라면 \(G_4 = R_5 = -1\) 만 남습니다.

---

## 5. Continuing task (지속적 과제)에서의 Return

### 5.1 슬라이드 식 (\(T = \infty\))

\[
G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots
= \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
\]

- **\(\gamma\)**: **discount rate (할인율)**, \(0 \le \gamma \le 1\)  
- **멀리 있는 보상**에 \(\gamma^k\) 가 곱해져 **가중치가 줄어듦** (시간이 지날수록 \(\gamma^k \to 0\) if \(\gamma<1\)).

### 5.2 왜 Continuing에서는 할인을 쓰나?

1. **무한 합의 수렴**  
   보상이 \(|R| \le R_{\max}\) 로 **유계**이고 \(0 \le \gamma < 1\) 이면  
   \[
   |G_t| \le R_{\max}(1 + \gamma + \gamma^2 + \cdots) = \frac{R_{\max}}{1-\gamma} < \infty
   \]  
   즉 **\(G_t\)가 무한대로 커지지 않도록** 수학적으로 다루기 쉬워집니다.

2. **시간 선호 (preference for sooner rewards)**  
   같은 크기의 보상이라도 **더 빨리** 받는 쪽을 선호한다는 뜻으로 해석할 수 있습니다.

3. **불확실성**  
   먼 미래는 모델·환경 변화로 **덜 확실**하다는 의미를 \(\gamma < 1\)에 담기도 합니다.

### 5.3 \(\gamma = 0\) 과 \(\gamma = 1\) 극한

- **\(\gamma = 0\)**:  
  \[
  G_t = R_{t+1}
  \]  
  **다음 한 스텝 보상만** 중요 (근시안).

- **\(\gamma = 1\)** (Continuing에서):  
  \[
  G_t = R_{t+1} + R_{t+2} + \cdots
  \]  
  **할인 없는 무한합**. 보상이 **항상 양수**이거나 **누적이 발산**하면 \(G_t\)가 **무한대**가 될 수 있어, 이론·알고리즘에서 **문제**가 됩니다.

### 5.4 슬라이드 문장

> **For the continuing task, if \(\gamma = 1\) then \(G_t \to \infty\)**

**의미 (정확히 말하면)**:

- **모든** 경우가 아니라, 예를 들어 **매 스텝 양의 보상이 꾸준히** 들어오면 합이 **발산**해서 \(G_t\)가 (유한 값이 아니라) **무한**으로 가는 방향이 됩니다.  
- 또는 **발산하지 않아도**, 무한합의 **수렴 조건**을 깨기 쉬워 **수학적으로 불편**해집니다.  
- 그래서 Continuing에서는 보통 **\(0 \le \gamma < 1\)** 을 씁니다.

---

## 6. Episodic vs Continuing — Return 식 한눈에

| 구분 | Return \(G_t\) (슬라이드 스타일) | 특징 |
|------|----------------------------------|------|
| **Episodic** | \(G_t = \sum_{k=t+1}^{T} R_k\) (인덱스는 교재에 맞게) | 끝 \(T\)가 있음, **유한 합** |
| **Continuing** | \(G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}\) | **무한**이므로 \(\gamma < 1\)로 수렴을 자주 가정 |

**같은 기호 \(G_t\)** 이지만 **문제 종류에 따라 정의가 다름**을 반드시 구분해야 합니다.

---

## 7. \(G_t\) 와 정책·가치 함수 (다음 슬라이드 예고)

- **정책 \(\pi\)** 가 정해지면, 궤적 \(\{S_t,A_t,R_{t+1},\ldots\}\) 의 분포가 정해집니다.  
- **상태 가치**는 보통  
  \[
  v_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]
  \]  
  즉 “\(t\)에서 상태가 \(s\)일 때 **Return의 기댓값**”입니다.  
- 따라서 **6페이지의 \(G_t\)** 를 정의하지 않으면 **가치 함수**를 정의할 수 없습니다.

---

## 8. 보상 인덱스 \(R_t\) vs \(R_{t+1}\) (혼동 방지)

교재·슬라이드에 따라:

- **\(S_t, A_t\)** 를 고른 **다음**에 오는 보상을 **\(R_{t+1}\)** 로 쓰거나  
- **\(R_t\)** 로 쓰는 경우도 있습니다.

**중요한 것**:  
- **한 교재 안에서** 정의를 통일하면 되고,  
- **Return의 구조**(에피소드면 유한합, 지속이면 할인합)는 **동일**합니다.

---

## 9. 수치 예시 (Continuing, 할인)

\(\gamma = 0.9\), \(t\) 이후 보상이 \(R_{t+1}=1,\ R_{t+2}=2,\ R_{t+3}=0\) 이고 그 다음은 전부 0이라 가정:

\[
G_t = 1 + 0.9 \cdot 2 + 0.9^2 \cdot 0 = 1 + 1.8 = 2.8
\]

\(\gamma = 0.5\) 이고 \(R_{t+k}=1\) 이 **영원히** 계속된다면 (이론 예시):

\[
G_t = 1 + 0.5 + 0.5^2 + \cdots = \frac{1}{1-0.5} = 2
\]

(등비급수 합 공식 \(1 + \gamma + \gamma^2 + \cdots = 1/(1-\gamma)\) for \(|\gamma|<1\).)

---

## 10. Episodic에서 \(\gamma\) 를 쓰는 경우 (슬라이드 밖 보충)

슬라이드 6페이지는 Episodic을 **할인 없는 합**으로만 적었지만, **구현·교재**에서는 Episodic에도 **\(0 < \gamma \le 1\)** 을 넣기도 합니다:

\[
G_t = R_{t+1} + \gamma R_{t+2} + \cdots + \gamma^{T-t-1} R_T
\]

- \(\gamma = 1\) 이면 슬라이드와 동일한 **단순 합**  
- \(\gamma < 1\) 이면 **마지막 근처 보상**에 더 큰 가중(상대적으로 **초반**은 할인)

**시험**: 강의가 **슬라이드 그대로**만 다루었다면 Episodic은 **할인 없는 \(G_t\)** 로 답하면 됩니다.

## 11. 자주 나오는 질문 (FAQ)

**Q1. 왜 “누적 보상”이 목표인가?**  
한 스텝만 최적화하면 **나중에 큰 손해**를 볼 수 있음. Return은 **장기 성과**를 한 숫자로 나타냄.

**Q2. Episodic인데 \(\gamma < 1\) 이면?**  
가능함. 슬라이드는 **가장 단순한** Episodic 정의만 제시.

**Q3. \(\gamma = 1\) 인 Continuing은 왜 위험?**  
무한합이 **발산**하거나, 최소한 **수렴 논의**가 어려워짐.

**Q4. \(G_t\) 와 \(G_{t+1}\) 관계?**  
\[
G_t = R_{t+1} + \gamma G_{t+1} \quad \text{(Continuing, 할인)}
\]  
Episodic \(\gamma=1\) 이면 \(G_t = R_{t+1} + G_{t+1}\) (단, \(t+1\)이 종료 직전이면 \(G_{t+1}\) 정의 주의).  
이 재귀가 **Bellman 방정식**으로 이어집니다.

---

## 12. 6페이지 요약 (암기용)

1. **목표**: 장기 **누적(또는 할인 누적) 보상** 최대화.  
2. **Return \(G_t\)**: 시점 \(t\) **이후** 받는 보상들의 합.  
3. **Episodic**: \(G_t = R_{t+1} + \cdots + R_T\) (**유한**).  
4. **Continuing**: \(G_t = \sum_{k\ge 0} \gamma^k R_{t+k+1}\), **\(\gamma \in [0,1]\)**.  
5. **Continuing에서 \(\gamma = 1\)** 이면 **무한합**이 커져 \(G_t \to \infty\) 같은 **문제**가 생기기 쉬움 → 보통 **\(\gamma < 1\)**.
