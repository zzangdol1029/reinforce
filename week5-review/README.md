## 🎯 핵심 학습 내용

### 강화학습 요소 정의

#### 🤖 Agent (에이전트)
```
SDN 컨트롤러의 라우팅 결정 모듈
→ 매 시점마다 최적의 다음 라우터 선택
```

#### 🌍 State (상태)
```
(현재 노드, 목적지 노드, 링크 상태)
→ 예: (노드 0, 노드 5, 대역폭-지연 정보)
```

#### ⚡ Action (행동)
```
다음 라우터 선택
→ 현재 노드에서 인접한 모든 라우터 중 선택
```

#### 🎁 Reward (보상)
```
대역폭 + 지연시간 기반 점수
Reward = 0.5 × (BW/BW_max) + 0.5 × (1 - delay/delay_max)
```

### 알고리즘 핵심

#### Policy Evaluation (정책 평가)
```
현재 정책의 가치함수 V(s) 계산
Q-table을 통해 각 상태-행동 쌍의 기댓값 평가
```

#### Policy Control (정책 개선)
```
ε-Greedy 전략: 탐험과 활용의 균형
- 확률 ε: 무작위 행동 (탐험)
- 확률 (1-ε): 최적 행동 (활용)
```

#### Q-value 업데이트
```
Q(s,a) ← Q(s,a) + α[r + γ·max_a' Q(s',a') - Q(s,a)]

α: 학습률 (0.1)
γ: 할인율 (0.9)
r: 즉시 보상
max_a' Q(s',a'): 다음 상태의 최대 Q값
```

## 📊 실험 결과

### 성능 비교

| 메트릭 | Q-Learning | Shortest Path | 개선율 |
|--------|-----------|--------------|--------|
| **누적 보상** | 2.9167 | 2.0000 | **45.83%** |
| **총 대역폭** | 39 Mbps | 27 Mbps | 44.44% |
| **총 지연** | 17 ms | 15 ms | -13.33% |
| **경로 길이** | 4 홉 | 3 홉 | 33.33% |

### 선택된 경로 비교

**Q-Learning 경로**: 0 → 1 → 2 → 3 → 5
```
링크별 분석:
  0→1: BW=10, Delay=5ms,  보상=0.7083
  1→2: BW=12, Delay=2ms,  보상=0.9167
  2→3: BW=9,  Delay=6ms,  보상=0.6250
  3→5: BW=8,  Delay=4ms,  보상=0.6667
  ────────────────────────────────
  합계: BW=39, Delay=17ms, 보상=2.9167
```

**Shortest Path 경로**: 0 → 1 → 4 → 5
```
링크별 분석:
  0→1: BW=10, Delay=5ms, 보상=0.7083
  1→4: BW=7,  Delay=7ms, 보상=0.5000
  4→5: BW=10, Delay=3ms, 보상=0.7917
  ────────────────────────
  합계: BW=27, Delay=15ms, 보상=2.0000
```

## 💡 주요 특징

### ✅ 장점

1. **모델 불필요 (Model-Free)**
   - 네트워크 동작 모델을 미리 알 필요 없음
   - 경험으로부터 직접 학습

2. **동적 적응**
   - 변하는 트래픽에 실시간 대응
   - 새로운 링크 추가 시 자동 재학습

3. **수렴성 보장**
   - 충분한 학습 후 최적 정책 수렴
   - 이론적으로 증명됨

4. **확장성**
   - 다양한 형태로 확장 가능 (DQN, 다중 에이전트)
   - 다양한 네트워크에 적용 가능

### ❌ 한계

1. **학습 시간**
   - 초기 학습에 많은 에피소드 필요
   - 대규모 네트워크는 수렴 느림

2. **차원의 저주**
   - 상태 공간 크기 증가로 Q-table 크기 증가
   - 메모리 문제 (대규모 네트워크)

3. **탐험 효율성**
   - 무작위 탐험은 비효율적
   - 더 똑똑한 탐험 전략 필요
