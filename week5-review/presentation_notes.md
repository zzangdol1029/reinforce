### 3️⃣ Q-Learning이란? (1분)

**"Q-Learning이 무엇인지 간단히 설명하겠습니다."**

```
Q-Learning = 행동 기반의 강화학습

특징:
  1. 환경 모델이 필요 없음 (모델 프리)
  2. 경험으로부터 직접 학습
  3. 최적 정책 수렴 보장
```

**핵심 아이디어:**

에이전트가 네트워크 환경에서 다양한 경로를 시도해보면서, 
어떤 행동(다음 노드 선택)이 가장 좋은 보상을 주는지 학습합니다.

**수식:**

```
Q(s, a) ← Q(s, a) + α[r + γ·max Q(s', a') - Q(s, a)]

즉:
  새로운 Q값 = 기존 Q값 + 학습률 × (오차)
```

---

### 4️⃣ 라우팅 문제 정형화 (1분)

**"라우팅 문제를 강화학습으로 어떻게 표현할까요?"**

#### Agent (에이전트)
→ **SDN 컨트롤러** (라우팅 결정을 담당)

#### State (상태)
→ **현재 노드, 목적지 노드, 네트워크 상태**

```
예: "현재 라우터 1번에서 라우터 5번으로 가야하고,
     라우터 1의 대역폭은 10Mbps, 지연은 5ms"
```

#### Action (행동)
→ **다음 라우터 선택**

```
예: "라우터 1에서 선택 가능한 다음 노드: 2번, 3번, 4번"
```

#### Reward (보상)
→ **대역폭 + 지연시간 기반 점수**

```
보상 = 0.5 × (대역폭 점수) + 0.5 × (빠름 점수)

예:
  - 대역폭 12Mbps, 지연 2ms → 높은 보상 ✓
  - 대역폭 5Mbps, 지연 12ms → 낮은 보상 ✗
```

---

### 5️⃣ Policy Evaluation & Control (1분)

**"Q-Learning은 두 가지 핵심 과정을 반복합니다."**

#### Policy Evaluation (정책 평가)
```
Q-table을 통해 현재 정책의 가치를 평가

Q(s, a) ← 더 정확한 값으로 업데이트
```

#### Policy Control (정책 개선)
```
ε-Greedy 전략으로 정책 개선

확률 ε: 무작위로 탐험 (새로운 경로 시도)
확률 (1-ε): 최적으로 활용 (좋은 경로 선택)
```

**왜 이렇게 할까요?**

초기에는 탐험으로 다양한 경로를 배우고,
나중에는 배운 최적 경로를 활용하기 위해서입니다.

### 6️⃣ 실험 결과 (45초)

**"실제로 이 알고리즘을 구현해서 시뮬레이션을 돌렸습니다."**

```
비교 대상: Shortest Path (최단 경로) 알고리즘

결과:
  Q-Learning 보상: 2.92
  Shortest Path 보상: 2.00
  
  성능 개선: 45.83%
```

**왜 Q-Learning이 더 좋을까요?**

- Shortest Path: 경로 길이만 최소화 (지연 중심)
- Q-Learning: 대역폭도 함께 고려 (균형잡힌 최적화)

결과적으로 Q-Learning이 선택한 경로:
```
0 → 1 → 2 → 3 → 5

총 대역폭: 39 Mbps (충분함!)
총 지연: 17ms (약간 더 길지만 감수 가능)
종합 점수: 2.92/4.0 (매우 좋음!)
```

### 9️⃣ 장점 & 한계 (45초)

**"Q-Learning의 장점과 한계를 정리하면:"**

#### ✅ 장점

1. **모델 불필요**: 네트워크 동작을 정확히 모르고도 학습
2. **자동 적응**: 트래픽 변화에 실시간 대응
3. **수렴성 보장**: 충분한 학습 후 최적해 도달
4. **확장성**: 다양한 형태로 확장 가능 (DQN, 다중 에이전트)

#### ❌ 한계

1. **학습 시간**: 초기에 많은 에피소드 필요
2. **차원의 저주**: 대규모 네트워크는 상태 공간 폭발
3. **탐험 비효율**: 무작위 탐험보다 똑똑한 탐험 필요
4. **정책 불안정성**: 학습 중 정책이 불안정할 수 있음

## 🤔 예상 질문 & 답변

### Q1: 기존 라우팅 알고리즘과의 차이는?

**답변:**
```
기존: 정적 규칙 또는 최단 경로만 고려
Q-Learning: 대역폭, 지연, 네트워크 상태 등 다양한 요소 동시 고려
           동적 환경에 자동 적응
           
비유: 
  기존: 지도만 보고 가장 짧은 길 선택
  Q-Learning: 매번 가보면서 가장 빠른 길 배우기
```

### Q2: 학습 시간은 얼마나 걸리나?

**답변:**
```
우리 실험: 500 에피소드 ≈ 몇 초
실제 네트워크: 상태 공간에 따라 다름
  - 중소 네트워크: 몇 분~몇 시간
  - 대규모 네트워크: 며칠~몇 주
  
개선 방법: 함수 근사 (신경망) 사용하면 빠름
```

### Q3: 네트워크가 변하면 어떻게 되나?

**답변:**
```
장점: Q-Learning은 주기적으로 재학습 가능
      새로운 링크 추가 → 자동으로 재학습
      
즉, "한 번 학습 후 끝"이 아니라
"지속적으로 배우면서 적응"
```

### Q4: 실제 네트워크에 적용 가능한가?

**답변:**
```
가능하지만 고려사항:
  1. 상태 공간이 매우 크면 → 함수 근사 필요
  2. 실시간 성능 필요 → 사전 학습 후 배포
  3. 보안 고려 → 악의적 노드 검출 추가
  4. 복합 요구사항 → 다중 에이전트 적용

현재 진행 중인 연구들이 이 문제들 해결 중
```

### Q5: 왜 DQN을 안 쓰고 기본 Q-Learning을 썼나?

**답변:**
```
1. 교육적 명확성
   - 기본이 더 이해하기 쉬움
   - 강화학습 원리 명확함

2. 실제 성능
   - 이 문제 규모에서는 Q-Learning으로 충분
   - DQN은 오버스펙

3. 계산 효율성
   - 신경망 없어서 빠름
   - 실시간 적용 용이

4. 논문 요구사항
   - "DQN 등 신경망 제외" 지시
```

### Q6: 보상 함수를 어떻게 설계했나?

**답변:**
```
Reward = 0.5 × 대역폭_점수 + 0.5 × 속도_점수

선택 기준:
  - 대역폭: "패킷 손실 없이 전송 가능?"
  - 지연: "실시간 요구사항 만족?"
  
조정 가능: 어느 것이 더 중요한지에 따라
  - 스트리밍: 대역폭 중시 (w1=0.7, w2=0.3)
  - 온라인 게임: 지연 중시 (w1=0.3, w2=0.7)
```
