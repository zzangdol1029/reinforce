# 📘 Ch.01 Introduction (강화학습 개요)

## 🔍 핵심 개념 상세

### 1) 머신러닝 3분류
**머신러닝(Machine Learning)** 은 "데이터로부터 패턴을 학습하는 알고리즘의 총칭"으로, 데이터의 **레이블 유무**와 **학습 신호의 종류**에 따라 3가지로 구분됩니다.

- **지도학습 (Supervised Learning)**
  - 입력 x와 정답 레이블 y가 쌍으로 주어진 데이터 (x, y)에서 함수 f: x→y를 학습
  - 학습 신호: **정답(label)**
  - 예: 회귀(집값 예측), 분류(스팸 메일, 객체 탐지)
  - 알고리즘: 선형회귀, 결정트리, 신경망

- **비지도학습 (Unsupervised Learning)**
  - 정답 없이 입력 x만으로 데이터의 구조/패턴 발견
  - 학습 신호: **데이터 자체의 구조**
  - 예: 클러스터링(K-means), 차원축소(PCA)

- **강화학습 (Reinforcement Learning)**
  - 환경과 상호작용하며 **보상(reward)** 으로 학습
  - 학습 신호: **보상(스칼라 값)**
  - 데이터가 미리 주어지지 않고, 시행착오(trial-and-error)를 통해 직접 수집
  - 예: 게임 AI, 자율주행, 로봇 제어

### 2) 강화학습의 5요소

```
                Action (A_t)
       ┌────────────────────────┐
       ▼                         │
  ┌─────────┐               ┌────────────┐
  │  Agent  │               │Environment │
  │  (학습  │               │   (환경)   │
  │   주체) │               │            │
  └─────────┘               └────────────┘
       ▲                         │
       │ State(S_{t+1}), Reward(R_{t+1})
       └─────────────────────────┘
```

| 요소 | 정의 | 로봇 보행 예시 |
|------|------|---------------|
| **Agent** (학습 주체) | 의사결정자, 행동을 선택 | 로봇 제어기 |
| **Environment** (환경) | Agent와 상호작용하는 모든 것 | 도로, 장애물, 중력 |
| **State** (상태) | 환경의 현재 상황을 나타내는 정보 | 장애물 위치, 관절 각도 |
| **Action** (행동) | Agent가 선택할 수 있는 동작 | 다리 관절의 회전 각도/속도 |
| **Reward** (보상) | 행동의 결과로 받는 스칼라 평가값 | 이동 거리, 넘어지지 않은 시간 |

**목표**: 누적 보상의 기댓값(expected return)을 최대화하는 정책(policy)을 학습.

## ✏️ 예상 문제 정답 풀이

### **[Q1-1]** 지도/비지도/강화학습 비교

**모범답안:**

| 구분 | 학습 데이터 | 학습 방식 |
|------|-----------|----------|
| 지도학습 | 입력 + 정답 (labeled) | 정답과 예측의 오차 최소화 |
| 비지도학습 | 입력만 (unlabeled) | 데이터의 분포/구조 학습 |
| 강화학습 | 환경에서 직접 수집한 (s, a, r, s') | 보상의 기댓값 최대화 |

**핵심 차이**: 지도학습은 "정답"을 따라하지만, 강화학습은 **순차적 의사결정** 문제이며 한 행동이 미래의 모든 보상에 영향을 미친다는 점에서 본질적으로 다릅니다.

---

### **[Q1-2]** 로봇 보행 문제의 5요소 매핑

**모범답안:**
- **Agent**: 로봇 제어기 (control policy)
- **Environment**: 도로, 장애물, 중력 등 외부 세계
- **State**: 장애물의 위치, 로봇의 관절 각도/속도, 자세
- **Action**: 각 다리 관절의 이동 각도 또는 토크
- **Reward**: 이동한 거리(+), 넘어짐(-), 에너지 소모(-)

---

### **[Q1-3]** RL과 supervised learning의 본질적 차이

**모범답안:**
1. **학습 신호**: 지도학습은 정답(label)을 따라하지만, 강화학습은 평가성 신호인 **보상(reward)** 만 받음. 어떤 행동이 "정답"인지는 직접 알려주지 않음.
2. **순차성**: 강화학습은 **순차적 의사결정 문제(sequential decision-making)** 로, 현재 행동이 미래 상태/보상에 영향을 줌. 지도학습은 i.i.d. 가정 하에 각 샘플을 독립적으로 다룸.
3. **데이터 분포의 영향**: 강화학습은 정책에 따라 데이터 분포가 바뀜(non-stationary). 지도학습은 고정된 분포를 가정.

# 📗 Ch.02 Bandit Problem

## 🔍 핵심 개념 상세

### 1) Multi-armed Bandit 문제

**Bandit** = 슬롯머신. **Multi-armed bandit**은 N개의 슬롯머신이 각각 다른 승률을 가질 때, 정해진 횟수 안에 최대 코인을 얻는 문제입니다.

- 처음에는 각 머신의 승률을 모름
- 플레이를 통해 정보를 수집하면서 좋은 머신을 찾아야 함
- **State는 정의되지 않음** (가장 단순한 형태의 RL 문제)

### 2) Action Value (행동 가치)

각 행동 a에 대한 **보상의 기댓값**:

```
Q(a) = E[ R | A = a ]
```

예: 슬롯머신 a의 승률이 0.6, 보상 = 1코인이면 Q(a) = 0.6.

### 3) Sample Average와 Incremental Update

n번의 플레이 후 평균 보상:

```
Q_n = ( R_1 + R_2 + ... + R_n ) / n
```

**점진적(Incremental) 갱신식** (메모리 효율을 위해 모든 R을 저장하지 않음):

```
Q_n = Q_{n-1} + (1/n) · ( R_n − Q_{n-1} )
```

**유도:**

```
Q_n = (1/n) · Σ_{i=1..n} R_i
    = (1/n) · [ R_n + Σ_{i=1..n-1} R_i ]
    = (1/n) · [ R_n + (n-1) · Q_{n-1} ]
    = R_n / n + Q_{n-1} − Q_{n-1} / n
    = Q_{n-1} + (1/n) · ( R_n − Q_{n-1} )
```

여기서 (1/n)은 **학습률(learning rate)** 역할.

### 4) Exploitation vs Exploration

- **Exploitation (활용)**: 지금까지 학습한 Q를 토대로 가장 좋은 행동 선택 (greedy)
- **Exploration (탐색)**: 더 나은 행동을 발견하기 위해 무작위 행동 시도

**ε-greedy 정책**:
- 확률 ε로 random (exploration)
- 확률 (1-ε)로 greedy (exploitation)
- ε = 0 → 완전 greedy (exploitation only)
- ε = 1 → 완전 random (exploration only)
- 일반적으로 ε ≈ 0.1 정도 사용

### 5) Stationary vs Non-stationary

- **Stationary**: 슬롯머신의 승률이 고정. 표본평균(1/n) 사용 적절.
- **Non-stationary**: 승률이 시간에 따라 변함. 최신 데이터에 더 큰 가중치 필요.

**고정 step size α 사용:**

```
Q_n = Q_{n-1} + α · ( R_n − Q_{n-1} )
```

**전개:**

```
Q_n = (1−α) · Q_{n-1} + α · R_n
    = (1−α)^n · Q_0 + Σ_{i=1..n} α · (1−α)^(n−i) · R_i
```

가중치 α·(1−α)^(n−i)는 i가 작을수록(과거일수록) 지수적으로 감소 → 최신 보상에 더 큰 비중.

## ✏️ 예상 문제 정답 풀이

### **[Q2-1]** Action value 정의 + incremental 식 유도

**정의:**

```
Q(a) = E[ R | A = a ]
```

행동 a를 선택했을 때 받는 보상의 기댓값.

**표본평균 추정:**

```
Q_n = (1/n) · Σ_{i=1..n} R_i
```

**Incremental 유도** (위 4) 참고):

```
Q_n = Q_{n-1} + (1/n) · ( R_n − Q_{n-1} )
```

**의미**: "현재 추정치 + 학습률 × (새 관측값 − 현재 추정치)" 형태로, 모든 강화학습 갱신식의 기본 패턴입니다.

---

### **[Q2-2]** ε-greedy 정책

**설명**: 확률 ε로 무작위 행동(exploration), (1−ε)로 가장 큰 Q값을 가진 행동(exploitation)을 선택하는 정책.

- **ε = 0**: 완전한 greedy. 한 번 좋다고 판단된 행동만 계속 선택 → exploration 없음 → 더 좋은 행동을 놓칠 위험.
- **ε = 1**: 완전한 random. 학습한 정보를 전혀 활용 안 함 → 비효율.

**실용적 선택**: 초반에는 ε를 크게(탐색), 점차 작게(활용) 줄이는 **decaying ε-greedy**가 일반적.

---

### **[Q2-3]** Stationary vs Non-stationary

| 항목 | Stationary | Non-stationary |
|------|-----------|----------------|
| 환경 | 보상 분포 고정 | 보상 분포 변화 |
| 가중치 | 모든 보상에 동일(1/n) | 최신 보상에 더 큰 가중치(α) |
| 갱신식 | Q_n = Q_{n-1} + (1/n)(R_n − Q_{n-1}) | Q_n = Q_{n-1} + α(R_n − Q_{n-1}) |

**고정 α 사용 이유**: 비정상 환경에서는 과거 데이터가 현재와 무관할 수 있음. 1/n은 n이 커질수록 새 데이터의 영향이 작아져 변화 추종 불가. α=고정값이면 가중치가 지수적으로 감소하여 항상 최신 데이터를 잘 반영.

---

### **[Q2-4]** 수치 계산 (보상 [1, 0, 5], Q_0 = 0)

| 단계 | 계산 | 결과 |
|------|------|------|
| Q_1 | 0 + (1/1)(1 − 0) | **1** |
| Q_2 | 1 + (1/2)(0 − 1) = 1 − 0.5 | **0.5** |
| Q_3 | 0.5 + (1/3)(5 − 0.5) = 0.5 + 1.5 | **2.0** |

검증: (1 + 0 + 5) / 3 = 2.0 ✓

# 📕 Ch.03 Markov Decision Process (MDP) ★★★

## 🔍 핵심 개념 상세

### 1) Markov Property

**정의:**

```
Pr( S_{t+1} | S_0, S_1, ..., S_t ) = Pr( S_{t+1} | S_t )
```

**의미**: 현재 상태가 주어지면, 미래는 과거와 독립적. 즉, 현재 상태 S_t가 과거 모든 정보를 충분히 요약(sufficient statistic)함.

> "The future is independent of the past given the present."

**RL에서의 표현**:

```
Pr( R_{t+1} = r, S_{t+1} = s' | S_0, A_0, R_1, ..., S_t, A_t )
   = Pr( R_{t+1} = r, S_{t+1} = s' | S_t, A_t )
   = p( s', r | s, a )
```

### 2) MDP의 정의 (5튜플)

```
MDP = ( 𝒮, 𝒜, T, r, γ )
```

- 𝒮 : 상태 집합
- 𝒜 : 행동 집합
- T = p(s' | s, a) : 상태 전이 확률
- r(s, a, s') : 보상 함수
- γ : 할인율 (0 ≤ γ ≤ 1)

**One-step dynamics:**

```
p( s', r | s, a ) = Pr( S_{t+1} = s', R_{t+1} = r | S_t = s, A_t = a )
```

**상태 전이 확률:**

```
p( s' | s, a ) = Σ_r  p( s', r | s, a )
```

**기대 보상:**

```
r(s, a) = E[ R_{t+1} | S_t = s, A_t = a ]
        = Σ_{s', r}  r · p(s', r | s, a)

r(s, a, s') = E[ R_{t+1} | S_t = s, A_t = a, S_{t+1} = s' ]
```

### 3) Episodic vs Continuing Task

- **Episodic task**: 종결 상태(terminal state)에서 끝남. 예: 바둑, 게임, Grid World.
- **Continuing task**: 끝이 없음. 예: 재고관리, Recycling Robot.

### 4) Return (수익)

**정의**: 시점 t 이후 받게 될 보상의 합.

- **Episodic** (T = 종결 시점):

```
G_t = R_{t+1} + R_{t+2} + ... + R_T
```

- **Continuing** (T = ∞):

```
G_t = R_{t+1} + γ·R_{t+2} + γ²·R_{t+3} + ...
    = Σ_{k=0..∞} γ^k · R_{t+k+1}
```

**할인율 γ의 역할:**
- γ = 0 : 즉각적 보상만 고려 (myopic)
- γ → 1 : 먼 미래 보상도 동등하게 중요시
- γ = 1 + continuing task → G_t가 발산 (∞)
- 일반적으로 0.9 ~ 0.99 사용

### 5) Value Function

**State-value function** (정책 π 하에서 상태 s의 가치):

```
v_π(s) = E_π[ G_t | S_t = s ]
```

"상태 s에서 출발해 정책 π를 따랐을 때의 기대 누적 보상."

**Action-value function** (Q-function):

```
q_π(s, a) = E_π[ G_t | S_t = s, A_t = a ]
```

"상태 s에서 행동 a를 한 후, 이후 정책 π를 따랐을 때의 기대 누적 보상."

**State-value와 Action-value의 관계:**

```
v_π(s) = Σ_a  π(a | s) · q_π(s, a)

q_π(s, a) = Σ_{s'}  p(s' | s, a) · [ r(s, a, s') + γ · v_π(s') ]
```

### 6) Bellman Equation 유도 (가장 중요!)

**Step 1**: Return의 재귀 관계

```
G_t = R_{t+1} + γ·R_{t+2} + γ²·R_{t+3} + ...
    = R_{t+1} + γ · ( R_{t+2} + γ·R_{t+3} + ... )
    = R_{t+1} + γ · G_{t+1}
```

**Step 2**: 정의에 대입

```
v_π(s) = E_π[ G_t | S_t = s ]
       = E_π[ R_{t+1} + γ · G_{t+1} | S_t = s ]
       = E_π[ R_{t+1} | S_t = s ] + γ · E_π[ G_{t+1} | S_t = s ]
                         (← 기댓값의 선형성)
```

**Step 3**: 각 항 전개

- 첫째 항:
```
E_π[ R_{t+1} | S_t = s ] = Σ_{a, s'}  π(a|s) · p(s'|s, a) · r(s, a, s')
```

- 둘째 항:
```
E_π[ G_{t+1} | S_t = s ] = Σ_{a, s'}  π(a|s) · p(s'|s, a) · v_π(s')
```

**최종 Bellman Equation (state-value):**

```
┌──────────────────────────────────────────────────────────────┐
│  v_π(s) = Σ_{a, s'}  π(a|s) · p(s'|s, a) · [r(s,a,s') + γ·v_π(s')]  │
└──────────────────────────────────────────────────────────────┘
```

**Bellman Equation (action-value):**

```
q_π(s, a) = Σ_{s'}  p(s'|s, a) · [ r(s, a, s') + γ · Σ_{a'}  π(a'|s') · q_π(s', a') ]
```

### 7) Optimal Value Function & Policy

**Optimal state-value:**

```
v*(s) = max_π  v_π(s)
```

**Optimal action-value:**

```
q*(s, a) = max_π  q_π(s, a)
```

**Optimal action (deterministic):**

```
μ*(s) = argmax_a  q*(s, a)
```

**Optimal policy:**

```
π*(a|s) = 1   if a = μ*(s)
        = 0   otherwise
```

### 8) Bellman Optimality Equation

**State-value:**

```
v*(s) = max_a  Σ_{s'}  p(s'|s, a) · [ r(s, a, s') + γ · v*(s') ]
```

**Action-value:**

```
q*(s, a) = Σ_{s'}  p(s'|s, a) · [ r(s, a, s') + γ · max_{a'}  q*(s', a') ]
```

> **Bellman equation = 연립방정식 문제**
> **Bellman optimality equation = 최적화 문제** (max 때문에 비선형)

## ✏️ 예상 문제 정답 풀이

### **[Q3-1]** Markov Property

**수식:**

```
Pr( S_{t+1} | S_0, S_1, ..., S_t ) = Pr( S_{t+1} | S_t )
```

**의미**: 현재 상태가 미래에 대한 모든 정보를 충분히 담고 있어, 과거 정보는 추가로 필요 없음. → "Future ⊥ Past | Present"

---

### **[Q3-2]** Return 정의 + γ 필요 이유

- **Episodic**: G_t = R_{t+1} + R_{t+2} + ... + R_T
- **Continuing**: G_t = Σ_{k=0..∞} γ^k · R_{t+k+1}

**γ가 필요한 이유:**
1. **수렴성 보장**: continuing task에서 γ = 1이면 무한 합이 발산. γ < 1이어야 |R| ≤ R_max인 한 G_t는 R_max / (1−γ)로 유한.
2. **시간 선호 반영**: 가까운 보상이 먼 보상보다 더 중요하다는 사람의 선호와 부합.
3. **불확실성 반영**: 먼 미래는 불확실성이 크므로 가치를 낮추는 것이 자연스러움.

---

### **[Q3-3]** State-value vs Action-value 관계

**정의:**
- v_π(s) = E_π[ G_t | S_t = s ] : 상태 s에서 정책 π를 따를 때 기대 수익
- q_π(s, a) = E_π[ G_t | S_t = s, A_t = a ] : 상태 s에서 행동 a를 한 뒤 정책 π를 따를 때 기대 수익

**관계식:**

```
v_π(s) = Σ_a  π(a|s) · q_π(s, a)
```

**유도**: 전체 기댓값의 법칙(law of total expectation)에 의해

```
v_π(s) = E_π[ G_t | S_t = s ]
       = Σ_a  Pr(A_t = a | S_t = s) · E_π[ G_t | S_t = s, A_t = a ]
       = Σ_a  π(a|s) · q_π(s, a)
```

---

### **[Q3-4]** Bellman 방정식 유도 (★ 핵심)

**Step 1**: G_t = R_{t+1} + γ·G_{t+1}

```
G_t = R_{t+1} + γ·R_{t+2} + γ²·R_{t+3} + ...
    = R_{t+1} + γ ( R_{t+2} + γ·R_{t+3} + ... )
    = R_{t+1} + γ · G_{t+1}
```

**Step 2**: 정의에 대입 + 기댓값 선형성

```
v_π(s) = E_π[ G_t | S_t = s ]
       = E_π[ R_{t+1} | S_t = s ] + γ · E_π[ G_{t+1} | S_t = s ]
```

**Step 3**: 각 항을 Σ π·p로 전개

- E_π[ R_{t+1} | S_t = s ] = Σ_{a,s'} π(a|s) · p(s'|s,a) · r(s,a,s')
- E_π[ G_{t+1} | S_t = s ] = Σ_{a,s'} π(a|s) · p(s'|s,a) · v_π(s')
  (다시 v_π(s')의 정의를 사용)

**최종:**

```
┌──────────────────────────────────────────────────────────────┐
│  v_π(s) = Σ_{a,s'}  π(a|s) · p(s'|s,a) · [r(s,a,s') + γ·v_π(s')]  │
└──────────────────────────────────────────────────────────────┘
```

# 📙 Ch.04 Dynamic Programming ★★★

## 🔍 핵심 개념 상세

### 1) DP의 본질

**Dynamic Programming**: 복잡한 문제를 작은 부분 문제로 분해하고, 부분 문제의 결과를 **저장**하여 재사용하는 기법.

- **장점**: Bellman 방정식(연립방정식)을 직접 풀지 않고 반복적으로 근사
- **단점**:
  - **차원의 저주**: 상태/행동 공간이 커지면 계산량 폭증
  - **모델 필요**: p(s'|s,a), r(s,a,s')를 알아야 함 (model-based)

### 2) Iterative Bellman Equation

**핵심 아이디어**: 추정치 V_k(s)를 만들고, Bellman 방정식의 우변에 대입하여 새 추정치 V_{k+1}(s)를 계산. 반복하면 v_π(s)로 수렴.

```
V_{k+1}(s) = Σ_{a,s'}  π(a|s) · p(s'|s,a) · [ r(s,a,s') + γ · V_k(s') ]
```

이를 **bootstrapping**이라 부름: "다음 상태의 추정치로 현재 상태의 추정치를 갱신".

**수렴 보장**: γ < 1인 contraction mapping이므로, 어떤 V_0에서 시작해도 v_π로 수렴 (Banach fixed-point theorem).

### 3) Policy Evaluation 알고리즘

```
입력: 정책 π, threshold θ
초기화: V(s) = 0  for all s ∈ S^+   (S^+ = S ∪ {terminal})

반복:
    Δ ← 0
    for each s ∈ S:
        v ← V(s)
        V(s) ← Σ_{a,s'} π(a|s) · p(s'|s,a) · [r(s,a,s') + γ · V(s')]
        Δ ← max(Δ, |v − V(s)|)
종료 조건: Δ < θ

출력: V ≈ v_π
```

### 4) Policy Improvement Theorem

**Statement**: 정책 π와 π'(deterministic, μ'(s)에 의해 정의)에 대해, 모든 s에서

```
q_π( s, μ'(s) ) ≥ v_π(s)
```

이면, 모든 s에서

```
v_{π'}(s) ≥ v_π(s)
```

즉, "한 step만 다르게 행동해도 더 좋다"면 "전체 정책을 바꿔도 더 좋다".

**Greedy improvement**:

```
μ'(s) = argmax_a  q_π(s, a)
      = argmax_a  Σ_{s'} p(s'|s,a) · [ r(s,a,s') + γ · v_π(s') ]
```

### 5) Policy Iteration vs Value Iteration

#### Policy Iteration (정책 반복법)

```
초기화: 임의의 π
반복:
    1. Policy Evaluation: V를 수렴할 때까지 반복 갱신 (v_π 계산)
    2. Policy Improvement: π' = greedy(V)
    3. π == π'이면 종료, 아니면 π ← π'
```

#### Value Iteration (가치 반복법)

```
초기화: V(s) = 0
반복:
    for each s:
        V(s) ← max_a  Σ_{s'} p(s'|s,a) · [r(s,a,s') + γ · V(s')]
종료: ΔV < threshold
출력: π* = greedy(V)
```

#### 비교표

| 항목 | Policy Iteration | Value Iteration |
|------|-----------------|-----------------|
| 평가 방식 | 수렴까지 반복 | 1 step만 |
| 개선 방식 | greedy | (max 통합되어 있음) |
| 정책 필요 | 필요 (평가 단계에서) | 불필요 (max로 대체) |
| 한 iteration 비용 | 큼 (평가 수렴) | 작음 |
| 수렴까지 iteration | 적음 | 많음 |
| Bellman 식 | v_π Bellman equation | Bellman optimality equation |

## ✏️ 예상 문제 정답 풀이

### **[Q4-1]** Iterative Bellman + Bootstrapping

**Bellman 직접 풀이의 어려움:**
- |S|개 미지수의 1차 연립방정식 → |S|³ 행렬 연산 필요
- 환경 모델 정확히 알아야 함
- 상태 공간 크면 비현실적

**Iterative Bellman 아이디어:**

```
V_{k+1}(s) = Σ_{a,s'}  π(a|s) · p(s'|s,a) · [ r + γ · V_k(s') ]
```

- V_0에서 시작해 반복 적용 → V_k → v_π
- 행렬 역연산 없이 단순 곱·합 반복 → 큰 문제에도 적용 가능

**Bootstrapping**: "다음 상태의 추정치 V_k(s')로 현재 상태의 추정치 V_{k+1}(s)를 갱신". 즉 자기 자신의 추정치로 자기 자신을 개선하는 방식.

### **[Q4-2]** Iterative Policy Evaluation 의사코드

```
Input: policy π, threshold θ > 0
Initialize V(s) = 0 for all s ∈ S^+   (V(terminal) = 0)

Repeat:
    Δ ← 0
    For each s ∈ S:
        v ← V(s)
        V(s) ← Σ_{a,s'}  π(a|s) · p(s'|s,a) · [r(s,a,s') + γ · V(s')]
        Δ ← max(Δ, |v − V(s)|)
Until Δ < θ

Output: V ≈ v_π
```

**구현 팁:** in-place update (한 배열 사용)가 별도 배열 사용보다 일반적으로 더 빨리 수렴.

### **[Q4-3]** Policy Iteration vs Value Iteration

(위 비교표 참고)

**핵심 차이:**
1. **평가 깊이**: PI는 수렴까지, VI는 1 step만.
2. **정책 명시성**: PI는 π를 명시적으로 유지, VI는 V만 유지하고 마지막에 greedy로 π 추출.
3. **사용 식**: PI는 Bellman equation, VI는 Bellman **optimality** equation.
4. **수렴 속도**: 일반적으로 VI가 단순하고 빠름.

### **[Q4-4]** Policy Improvement Theorem

**Statement:**
> 두 결정적 정책 π, π'에 대해 모든 s ∈ S에서
> q_π(s, μ'(s)) ≥ v_π(s) 이면, 모든 s에서 v_{π'}(s) ≥ v_π(s).

**보장하는 바**: greedy improvement (μ'(s) = argmax_a q_π(s,a))는 항상 정책을 개선하거나 동일하게 유지함. 즉, **monotonic improvement** 보장.

**Optimal에 도달**: 더 이상 개선이 없으면(π' = π), Bellman optimality equation을 만족 → π = π*.

### **[Q4-5]** Grid World 손계산 예시

**(b) Value Iteration 1-step (2-Grid World, γ = 0.9, V_0 = 0):**

```
V_1(L1) = max{ r(L1, left, L1) + 0.9 · V_0(L1),
               r(L1, right, L2) + 0.9 · V_0(L2) }
        = max{ -1 + 0,  1 + 0 }
        = 1

V_1(L2) = max{ 0 + 0,  -1 + 0 }
        = 0
```

(보상 정의: L1→L1 left = -1, L1→L2 right = +1, L2→L1 left = 0, L2→L2 right = -1 가정)

**다음 step:**

```
V_2(L1) = max{ -1 + 0.9·(1),  1 + 0.9·(0) } = max{ -0.1, 1 } = 1
V_2(L2) = max{  0 + 0.9·(1), -1 + 0.9·(0) } = max{  0.9,-1 } = 0.9
```

수렴 시: v*(L2) = 0.9 · v*(L1), v*(L1) = 1 + 0.9 · v*(L2)
→ v*(L1) ≈ 5.26, v*(L2) ≈ 4.74.

### **[Q4-6]** DP의 한계

1. **차원의 저주(Curse of Dimensionality)**: 상태 수 |S|가 커질 때 계산량과 메모리가 |S|에 비례하여 폭증. 예: 바둑(|S| ≈ 10^170) 같은 문제에 직접 적용 불가.
2. **환경 모델 필요(Model-based)**: p(s'|s,a), r(s,a,s')를 미리 알아야 함. 현실 문제에서 이 모델을 정확히 아는 경우는 거의 없음. → MC, TD 같은 model-free 방법이 필요한 이유.

# 📓 Ch.05 Monte Carlo Method ★★

## 🔍 핵심 개념 상세

### 1) Monte Carlo의 정의

**Monte Carlo Method**: 무작위 샘플링을 반복하여 수치적으로 결과를 추정하는 알고리즘.

**예 1) π 계산**: 단위 정사각형에 무작위 점 N개 → 사분원 안의 점 비율 × 4 → π 추정.

**예 2) 주사위 두 개의 합 기댓값**:
- 분포모델: P(X = k)를 직접 계산 → 7.0
- 샘플모델: 주사위를 N번 던져 평균 → 7.0에 근사

### 2) 분포모델 vs 샘플모델

- **분포모델 (Distribution model)**: 확률 분포를 명시적으로 알고 있음 → 기댓값을 해석적으로 계산.
- **샘플모델 (Sample model)**: 샘플을 생성할 수 있을 뿐, 분포는 모름 → 평균으로 기댓값 추정.

→ 강화학습에서 환경 모델을 모를 때 샘플모델을 쓸 수 있다는 점에서 MC가 유용.

### 3) MC Prediction (Policy Evaluation)

**아이디어**: v_π(s) = E_π[ G_t | S_t = s ] → 정책 π를 따라 에피소드를 여러 번 진행하고, s 방문 후의 G 평균.

**알고리즘:**

```
Input: policy π
Initialize: V(s) = 0, Returns(s) = empty list

For each episode:
    Generate episode (S_0, A_0, R_1, ..., S_T) following π
    G ← 0
    For t = T-1, T-2, ..., 0:
        G ← γ · G + R_{t+1}
        Append G to Returns(S_t)
        V(S_t) ← average(Returns(S_t))
```

**Incremental 형태:**

```
V_n(s) = V_{n-1}(s) + (1/n) · [ G^(n) − V_{n-1}(s) ]
```

### 4) MC vs DP

- **DP**: bootstrap (V_{k+1}(s)는 V_k(s')에 의존), 모델 필요.
- **MC**: 실제 episode의 G를 그대로 사용 (no bootstrap), 모델 불필요. **Learning** (실제 경험으로 학습) vs **Computing** (DP는 모델로 계산).

### 5) MC Control (Policy Improvement)

**Q를 추정해야 하는 이유**: V만 알면 정책 개선 시

```
μ(s) = argmax_a  Σ_{s'} p(s'|s,a) · [ r + γ · V(s') ]
```

처럼 모델 p가 필요함. 반면 Q를 직접 추정하면

```
μ(s) = argmax_a  Q(s, a)
```

로 모델 없이 개선 가능.

**ε-greedy 사용 이유**: greedy만 쓰면 한 번도 안 가본 (s, a) 쌍을 영원히 안 가서 Q 추정 불가능 → exploration 보장 필요.

### 6) Exponential Moving Average

1/n 대신 고정 α 사용:

```
Q_n(s, a) = Q_{n-1}(s, a) + α · [ G^(n) − Q_{n-1}(s, a) ]
```

전개하면 G^(i)의 가중치가 α·(1−α)^(n−i) → 최신 데이터에 더 큰 가중치. 비정상 환경 또는 학습 초기 부정확한 Q를 빠르게 잊고 갱신할 때 유리.

### **[Q5-1]** Monte Carlo 개념

**개념**: 무작위 샘플링을 반복해 분포의 통계량을 추정하는 기법.

### **[Q5-3]** Incremental 갱신 유도

```
V_n = (1/n) · Σ_{i=1..n} G^(i)
    = (1/n) · [ G^(n) + Σ_{i=1..n-1} G^(i) ]
    = (1/n) · [ G^(n) + (n − 1) · V_{n-1} ]
    = G^(n) / n  +  V_{n-1}  −  V_{n-1} / n
    = V_{n-1} + (1/n) · ( G^(n) − V_{n-1} )
```

```
┌──────────────────────────────────────────────┐
│  V_n(s) = V_{n-1}(s) + (1/n) · [ G^(n) − V_{n-1}(s) ]  │
└──────────────────────────────────────────────┘
```

---

### **[Q5-4]** Q를 추정해야 하는 이유

V를 사용하면 정책 개선 시

```
μ(s) = argmax_a  Σ_{s'} p(s'|s, a) · [ r + γ · V(s') ]
```

이 필요하므로 **환경 모델 p, r을 알아야 함**.

Q를 직접 추정하면

```
μ(s) = argmax_a  Q(s, a)
```

로 모델 없이 개선 가능 → MC, TD 같은 model-free 방법이 V 대신 Q를 사용하는 이유.

---

### **[Q5-5]** EMA의 장점과 가중치 변화

**식**: Q_n = Q_{n-1} + α · ( G^(n) − Q_{n-1} )

**가중치 전개**:

```
Q_n = (1 − α) · Q_{n-1} + α · G^(n)
    = (1 − α)^n · Q_0 + Σ_{i=1..n} α · (1 − α)^(n−i) · G^(i)
```

가중치 α·(1−α)^(n−i)는 i가 작을수록(과거) 지수적으로 감소 → 최신 데이터에 큰 가중치.

**장점:**
1. 비정상 환경에서 변화 추종 가능
2. 초기의 부정확한 Q를 빠르게 잊음
3. 메모리 일정 (n 무관)

**단점:** stationary 환경에서 표본평균보다 분산 큼.

---

### **[Q5-6]** ε, α 변화의 영향

- **ε ↑**: 탐색 ↑ → 다양한 (s, a) 경험 → 학습 안정적이지만 보상은 낮을 수 있음
- **ε ↓**: 활용 ↑ → 빠른 수렴이지만 local optimum 위험
- **α ↑**: 최신 데이터 가중 ↑ → 빠른 갱신이지만 분산 큼
- **α ↓**: 과거 평균에 가까움 → 안정적이지만 변화 느림

5x5 Grid World에서는 보통 ε ≈ 0.1, α ≈ 0.1 정도가 균형점.

# 📔 Ch.06 Temporal Difference Method ★★★

## 🔍 핵심 개념 상세

### 1) TD = MC + DP

| 방법 | Bootstrapping | Sampling |
|------|--------------|----------|
| DP | O (V_k(s')로 V_{k+1}(s) 계산) | X (모델로 모든 s' 고려) |
| MC | X (실제 G 사용) | O (샘플 episode) |
| **TD** | **O** (V(S_{t+1}) 사용) | **O** (실제 transition 샘플) |

→ TD는 두 방법의 장점을 결합:
- 모델 불필요 (MC처럼)
- 매 step 갱신 가능, 지속성 과제 가능 (DP처럼)
- 분산 작음 (MC보다)

### 2) TD(0) 갱신식

**유도**: Bellman 방정식

```
v_π(s) = E[ R_{t+1} + γ · v_π(S_{t+1}) | S_t = s ]
```

이 기댓값을 한 샘플로 근사: R_{t+1} + γ · V(S_{t+1}) (TD target).

갱신:

```
┌─────────────────────────────────────────────────────────────────┐
│  V(S_t) ← V(S_t) + α · [ R_{t+1} + γ · V(S_{t+1}) − V(S_t) ]    │
│                       └────────── TD target ──────────┘          │
│                       └──────────── TD error δ_t ──────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

**TD error:**

```
δ_t = R_{t+1} + γ · V(S_{t+1}) − V(S_t)
```

### 3) TD vs MC 비교

| 항목 | MC | TD |
|------|-----|-----|
| 갱신 시점 | 에피소드 종료 후 | 매 step (1-step) |
| 사용 정보 | 실제 G (전체 미래) | R + γ·V(S') (1-step + bootstrap) |
| 분산 | 큼 (G가 긴 합) | 작음 (1-step만) |
| 편향 | 작음 (실제 값) | 있음 (V 추정치 사용) |
| 지속성 과제 | X | O |

### 4) SARSA (On-policy TD Control)

**5튜플** (S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})로 갱신:

```
Q(S_t, A_t) ← Q(S_t, A_t) + α · [ R_{t+1} + γ · Q(S_{t+1}, A_{t+1}) − Q(S_t, A_t) ]
```

**알고리즘:**

```
Initialize Q(s, a) arbitrarily

For each episode:
    S ← initial state
    A ← ε-greedy(Q, S)
    Repeat (for each step):
        Take A, observe R, S'
        A' ← ε-greedy(Q, S')
        Q(S, A) ← Q(S, A) + α · [ R + γ · Q(S', A') − Q(S, A) ]
        S ← S' ; A ← A'
    until S is terminal
```

**On-policy**: 행동을 선택한 정책(ε-greedy)과 평가/개선하는 정책이 동일.

### 5) On-policy vs Off-policy

- **On-policy**: 행동 정책(behavior policy) = 대상 정책(target policy)
  - 예: SARSA, MC Control
  - "내가 직접 한 경험으로 내 정책 개선"
- **Off-policy**: 행동 정책 ≠ 대상 정책
  - 예: Q-Learning, Off-policy SARSA
  - "다른 정책의 경험으로 내 정책 개선"
  - 장점: 데이터 재사용, exploration과 exploitation 분리, replay buffer 사용 가능

### 6) Importance Sampling (IS)

**문제**: 분포 b에서 샘플링한 데이터로 분포 π의 기댓값을 추정.

```
E_π[x] = Σ_x  π(x) · x
       = Σ_x  b(x) · ( π(x) / b(x) ) · x
       = E_b[ ρ(x) · x ]
```

**Importance ratio**: ρ(x) = π(x) / b(x)

**MC 추정**:

```
E_π[x] ≈ (1/n) · Σ_{i=1..n} ρ(x^(i)) · x^(i),    x^(i) ~ b
```

**문제점**: π와 b가 많이 다르면 ρ가 매우 커지거나 작아져서 추정의 분산이 폭발.

### 7) Off-policy SARSA

**Target policy** π : greedy
**Behavior policy** b : ε-greedy

```
Q(S_t, A_t) ← Q(S_t, A_t) + α · [ R + γ · ρ · Q(S_{t+1}, A_{t+1}) − Q(S_t, A_t) ]
```

여기서 ρ = π(A_{t+1} | S_{t+1}) / b(A_{t+1} | S_{t+1}).

### 8) Q-Learning

**Bellman optimality equation**의 sampling 버전:

```
┌──────────────────────────────────────────────────────────────────────┐
│  Q(S_t, A_t) ← Q(S_t, A_t) + α · [ R_{t+1} + γ · max_a Q(S_{t+1}, a) − Q(S_t, A_t) ]  │
└──────────────────────────────────────────────────────────────────────┘
```

**알고리즘:**

```
Initialize Q(s, a)

For each episode:
    S ← initial state
    Repeat:
        A ← ε-greedy(Q, S)
        Take A, observe R, S'
        Q(S, A) ← Q(S, A) + α · [ R + γ · max_a Q(S', a) − Q(S, A) ]
        S ← S'
    until S is terminal
```

**Off-policy인데 IS 불필요한 이유**: target에서 다음 행동을 b로부터 샘플링하지 않고 max로 직접 선택하므로, target에 b의 영향이 없음.

### 9) SARSA vs Q-Learning

| 항목 | SARSA | Q-Learning |
|------|--------|------------|
| 기반 식 | Bellman equation | Bellman optimality equation |
| Target | R + γ·Q(S', A') | R + γ·max_a Q(S', a) |
| 다음 행동 | 실제 샘플링한 A' 사용 | max로 가상의 최적 행동 |
| On/Off-policy | On | Off |
| IS 필요? | (on-policy) 불필요 | 불필요 (max 사용) |
| 학습 안정성 | 보수적 (실제 정책 반영) | 적극적 (greedy 학습) |
| 위험 회피 | O (실제 ε-greedy의 위험까지 학습) | X (greedy 가정으로 위험 무시) |

**유명한 예시 (Cliff walking)**: SARSA는 절벽에서 멀리 돌아가는 안전 경로를 학습, Q-Learning은 절벽 바로 옆 최단 경로를 학습 (실제로는 ε-greedy로 가끔 떨어짐).

### **[Q6-2]** TD(0) 갱신식 (★)

```
V(S_t) ← V(S_t) + α · [ R_{t+1} + γ · V(S_{t+1}) − V(S_t) ]
                     └────────── TD target ──────────┘
                     └──────────── TD error δ_t ──────────────┘
```

**MC와의 차이:**
- MC: V(S_t) ← V(S_t) + α · [ G_t − V(S_t) ]   (G_t는 episode 끝까지 누적)
- TD: V(S_t) ← V(S_t) + α · [ R_{t+1} + γ · V(S_{t+1}) − V(S_t) ]   (1 step + bootstrap)

→ TD는 1-step만 보고 다음 상태의 추정치 V(S_{t+1})로 나머지를 대체. 그래서 매 step 갱신 가능 (지속성 과제 가능)이고 분산이 작음.

---

### **[Q6-3]** SARSA 알고리즘

```
Input: ε ∈ (0,1), α ∈ (0,1], γ ∈ (0,1]
Initialize Q(s, a) arbitrarily, Q(terminal, ·) = 0

For each episode:
    Initialize S
    Choose A ← ε-greedy(Q, S)
    For each step:
        Take action A, observe R, S'
        Choose A' ← ε-greedy(Q, S')
        Q(S, A) ← Q(S, A) + α · [ R + γ · Q(S', A') − Q(S, A) ]
        S ← S' ; A ← A'
    until S is terminal
```

**이름의 어원**: 갱신에 사용하는 5튜플 (**S**tate, **A**ction, **R**eward, **S**tate', **A**ction')의 머리글자.

---

### **[Q6-4]** On-policy vs Off-policy (★)

- **On-policy**: 행동을 결정하는 정책(behavior)과 학습하는 정책(target)이 **같음**.
  - 예: SARSA — ε-greedy로 행동하고 ε-greedy를 평가/개선.
  - 예: MC Control (ε-soft)
- **Off-policy**: behavior와 target이 **다름**.
  - 예: Q-Learning — ε-greedy로 행동하지만 greedy 정책을 학습.
  - 예: Off-policy SARSA — behavior = ε-greedy, target = greedy
  - 장점: 옛 데이터 재활용 (replay buffer), 다른 사람/시뮬레이터의 데이터 활용 가능, 탐색-활용 분리.

---

### **[Q6-5]** Importance Sampling

**식:**

```
E_π[X] = E_b[ ( π(X) / b(X) ) · X ] = E_b[ ρ(X) · X ]
```

**Off-policy SARSA에서 ρ가 곱해지는 위치**: TD target의 next-action 부분.

```
Q(S_t, A_t) ← Q(S_t, A_t) + α · [ R_{t+1} + γ · ρ · Q(S_{t+1}, A_{t+1}) − Q(S_t, A_t) ]
```

여기서 ρ = π(A_{t+1} | S_{t+1}) / b(A_{t+1} | S_{t+1}).

이는 "behavior policy로 샘플링한 A_{t+1}을 target policy 분포의 기댓값으로 보정"하는 역할.

**문제점**: π와 b가 많이 다르면 ρ가 매우 크거나 작아져 분산 폭발 → 학습 불안정.

---

### **[Q6-6]** SARSA vs Q-Learning 5가지 관점 비교 (★ 핵심)

| 관점 | SARSA | Q-Learning |
|------|--------|------------|
| (1) Bellman 식 | Bellman equation | Bellman **optimality** equation |
| (2) Target | R + γ·Q(S', A') | R + γ·max_a Q(S', a) |
| (3) On/Off-policy | On-policy | Off-policy |
| (4) IS 필요 | 불필요 (on-policy) | 불필요 (max 사용) |
| (5) 학습 특성 | 실제 정책 반영, 보수적·안전 | greedy 가정, 적극적·최적 |

**Cliff walking 직관**: SARSA는 ε-greedy의 무작위성이 절벽 추락 위험을 학습에 반영하므로 안전 경로 선호. Q-Learning은 max만 보므로 최단 경로 학습하지만 학습 중에는 ε-greedy로 자주 떨어짐.

---

### **[Q6-7]** Q-Learning이 IS 불필요한 이유

Q-Learning의 target:

```
R_{t+1} + γ · max_a Q(S_{t+1}, a)
```

여기서 다음 행동을 behavior policy b로 **샘플링**하는 게 아니라, Q를 통해 **결정론적으로 max** 선택. 즉, target 계산에 b의 분포가 등장하지 않으므로 분포 보정(IS)이 불필요.

(반면 off-policy SARSA는 target에 Q(S_{t+1}, A_{t+1})이 있고, 여기서 A_{t+1} ~ b이므로 π 분포로 보정 필요.)

→ Q-Learning은 **off-policy의 장점(데이터 재활용 등)** 을 누리면서 **IS의 단점(분산 폭발)** 을 피하므로 매우 안정적이고 강력. DQN의 기반이 됨.

---

### **[Q6-8]** 5x5 Grid World Q-Learning 결과

학습 후 각 상태의 V(s) = max_a Q(s, a)는 보상(+1, -1)의 위치를 향하는 방향으로 그라디언트를 형성하고, 정책 π*는 각 상태에서 최대 Q를 가진 화살표(↑↓←→)로 표시됨.

(γ = 0.9, ε = 0.1 정도로 충분한 episode를 학습하면 +1 칸 인접 상태들은 V ≈ 0.9, 그 다음은 0.81, … 같은 식으로 지수 감소.)
